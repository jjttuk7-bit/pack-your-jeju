# 2026 제주 공공데이터·AI 창업경진대회용 데이터 업그레이드 전략

작성일: 2026-07-11  
대상 문서: `2026년 제주 공공데이터·AI 활용 창업경진대회 공고문 (2).hwpx`  
대상 제품: 제주를 담다 / Pack Your Jeju

## 1. 결론

이번 경진대회 버전은 기존 여행 추천 서비스가 아니라 **제주 공공데이터와 AI를 결합한 여행 의사결정 신뢰 인프라**로 재정의해야 한다.

현재 서비스는 이미 비짓제주 관광 데이터, 위생등급, 주차장, 버스정류장, 콘텐츠 수정요청, 기상 신호를 구조화했거나 일부 기능으로 사용하고 있다. 따라서 우승용 업그레이드의 핵심은 “P0 데이터를 새로 붙이는 것”이 아니라, **이미 구조화된 데이터를 경진대회 심사위원이 바로 이해할 수 있는 완성 기능·시연·사업계획서 근거로 끌어올리는 것**이다.

1. 제주 공공데이터 활용도가 높다.
2. AI가 공공데이터를 실제 의사결정으로 바꾼다.
3. 제주 사회 문제 해결과 사업화 가능성이 동시에 보인다.

따라서 데이터 전략은 다음처럼 재분류한다.

| 구분 | 데이터 축 | 현재 상태 | 대회용 남은 일 |
|---|---|---|
| 이미 구조화/부분 구현 | 기상·기후 데이터 | 기상청 단기예보/초단기예보 파서와 날씨 신호 있음 | 특보·리스크 재배치 시연 강화 |
| 이미 구조화/부분 구현 | 교통·주차·정류장 접근성 | `transit_point`, 주차장/정류장 반경 판정 있음 | 노선·도착·교통약자 관점 보강 |
| 이미 구조화/부분 구현 | 축제·숙박·쇼핑·문화 비짓제주 확장 | `category`와 순간 카드 12종으로 연결됨 | 날짜 매칭·동선 기준점·시연 문구 강화 |
| 이미 구조화/구현 | 콘텐츠 수정요청 기반 신뢰 검증 | `fix_request_detail`, `has_fix_request`, `/verify` 판정 있음 | 발표용 대표 사례와 전후 비교 장면 구성 |
| 추가 후보 | 관광 혼잡·방문/상권 데이터 | 아직 핵심 구조 아님 | 지역 분산·사회적 가치 점수용 신규 축 |
| 추가 후보 | 무장애·안전·편의시설 데이터 | 일부 companion caution만 있음 | 부모님/아이/교통약자 시나리오 강화 |
| 추가 후보 | JDC/제주개발공사 관련 데이터 | 아직 확인 필요 | 기관 데이터 가점과 사업화 서사용 |

## 2. 공고문 기준으로 본 승리 조건

공고문은 공공데이터와 AI를 모두 활용해야 한다고 명시한다. 특히 제품·서비스 개발 부문은 신청 마감일까지 시제품이 완료되어야 하며, 2차 발표평가에서 서비스 완성도와 사업화 가능성을 실제 시연으로 보여줘야 한다.

또한 전국대회 본선 심사에 AI 기술 활용 및 AI 서비스 평가지표가 신설되어, 지역 예선 단계부터 AI 기술 활용도가 평가항목에 반영된다. 각 주최·주관 기관별 공공데이터 활용 시 가점 2점이 부여되고, 기상기후 데이터 활용 특별상도 있다.

이 조건을 제품 전략으로 바꾸면 다음과 같다.

- 단순 추천보다 **검증·판정·대체안 제시**가 강해야 한다.
- “AI가 답했다”보다 **AI가 어떤 공공데이터를 근거로 판단했는지**가 보여야 한다.
- 발표에서는 장소 카드보다 **데이터가 의사결정을 바꾸는 순간**을 보여줘야 한다.
- 기상청 데이터는 선택이 아니라 사실상 필수다.
- 제주특별자치도, 제주지방기상청, JDC, 제주특별자치도개발공사 관련 데이터를 하나라도 더 연결하면 가점 서사가 생긴다.

## 3. 현재 서비스에 이미 있는 데이터 자산

현재 프로젝트는 다음 데이터를 이미 사용하거나 사용할 수 있는 구조를 갖고 있다.

| 데이터 | 현재 쓰임 | 강점 | 보완점 |
|---|---|---|---|
| 비짓제주 관광정보 | 장소, 음식, 카페, 숙박, 축제, 쇼핑, 문화 후보 | 제주 특화성과 설명력이 좋음 | 카테고리별 시연 장면을 더 선명하게 만들어야 함 |
| 콘텐츠 수정요청 CSV | 정보 변경·폐업·주의 신호 | “AI가 틀린 정보를 막는다”는 차별성 | 발표에서 수정요청 건수와 사례를 더 강하게 제시 |
| 위생등급/모범음식점 | 음식점 신뢰 보강 | 맛집 추천의 공공데이터 근거 | 음식점만이 아니라 “안심 소비” 메시지로 확장 |
| 주차장 데이터 | 장소 접근성 배지 | 렌터카 여행자에게 실용적 | 실시간/혼잡 여부가 없으면 “존재 확인”으로만 표현 |
| 버스정류장 데이터 | 대중교통 접근성 배지 | 비렌터카 여행자 배려 | 노선/도착 정보까지 붙이면 시연력이 커짐 |
| 기상청 예보 스냅샷 | 여행 기간 날씨 신호 | 특별상과 직접 연결 | 날씨 리스크 기반 일정 재배치로 업그레이드 필요 |
| 웹검색 출처 도구 | 최신 정보 보강 | 최신성 한계 보완 | 공공데이터 검증과 웹 출처 기준을 UI에서 구분해야 함 |

## 4. 데이터별 현재 상태와 업그레이드 방향

### 4.1 기상·기후 데이터

이미 구조화되어 있다. 현재 코드에는 기상청 API허브 단기예보와 공공데이터포털 단기예보 JSON을 파싱하는 구조가 있고, 강수확률·하늘상태·기온·풍속·위험 신호를 여행 날씨 신호로 정규화한다.

따라서 “추가 데이터”가 아니라 “경진대회용 기능 고도화”로 봐야 한다. 이 대회에는 제주지방기상청장 특별상이 있고, 공고문에도 기상기후 데이터 활용 특별상 조건이 명시되어 있으므로, 지금 있는 날씨 신호를 일정 재배치와 시연 장면으로 끌어올리는 것이 중요하다.

추천 데이터:

- 기상청 단기예보/초단기예보
- 강수확률, 하늘상태, 기온, 풍속
- 특보/주의보: 강풍, 호우, 폭염, 풍랑
- 해양/해변 관련 관측 또는 예보

제품 적용:

- 야외 순간 카드 위험도 산출: 오름, 바다 산책, 노을, 곶자왈
- 실내 대체 일정 자동 제안: 카페, 문화, 쇼핑, 숙박 주변 일정
- “비 오는 2박 3일 제주” 시연 시나리오
- 하루방 답변에서 “날씨 때문에 이 일정을 뒤로 미루는 이유” 설명

대회 메시지:

> 제주 여행 실패의 가장 큰 변수인 날씨를 공공데이터로 사전에 감지하고, AI가 여행자의 일정 선택을 안전하게 바꿉니다.

현재 상태: 구조화/부분 구현 완료

남은 일:

- 특보/주의보까지 붙일지 결정
- 야외 순간 카드별 날씨 민감도 테이블 만들기
- 비·강풍·폭염 시 실내 대체 코스 자동 설명 강화
- 발표에서 “기상청 데이터가 일정을 바꾼다”는 장면으로 시연

참고 출처:

- 기상청 API허브: https://apihub.kma.go.kr/
- 공공데이터포털: https://www.data.go.kr/

### 4.2 교통·접근성 데이터

이미 구조화되어 있다. 현재 스키마에는 `transit_point(kind='parking'|'busstop')`가 있고, 장소 좌표 기준으로 주차장 1km, 정류장 500m 반경 접근성 판정을 한다. 하루방도 특정 장소의 주차장·정류소 질문에 `get_place_detail` 결과의 transit만 근거로 답하도록 되어 있다.

따라서 기본 접근성 데이터는 이미 반영된 상태다. 우승용으로는 “존재 확인”을 넘어 “갈 수 있는 여행”을 보여주는 설명과 시연을 강화해야 한다.

추천 데이터:

- 버스정류장 위치
- 버스 노선/운행 정보
- 버스 도착 정보 또는 정류장별 접근성 데이터
- 공영주차장 위치, 주차면수, 가능하면 실시간 혼잡도
- 전기차 충전소 위치 및 이용 가능 상태
- 공항/항만/렌터카/환승 거점 데이터

제품 적용:

- 렌터카 없음/부모님 동행/아이 동반 조건에서 접근성 점수 계산
- “공항 근처 마지막 날 3시간 코스” 자동 조립
- 장소 카드에 `차로 쉬움`, `버스로 가능`, `도보 부담`, `확인 필요` 표시
- 교통 약자에게 무리한 오름·해변 후보를 낮은 우선순위로 조정

대회 메시지:

> 추천을 넘어, 실제로 갈 수 있는지를 공공 교통·주차 데이터로 검증합니다.

현재 상태: 구조화/부분 구현 완료

남은 일:

- 버스 노선/도착 정보는 아직 부족하므로 P1 후보로 분리
- 부모님/아이 동반 조건에서 접근성 점수 설명 강화
- 공항 근처 마지막 날 코스에서 이동 부담을 더 명시
- 주차장/정류장 반경 기준을 사업계획서에 명확히 표기

참고 출처:

- 공공데이터포털 교통물류 분류: https://www.data.go.kr/
- 제주데이터허브: https://www.jejudatahub.net/

### 4.3 관광 혼잡·방문객·지역 분산 데이터

단순 관광 추천 앱과 차별화하려면 “제주 사회 문제 해결”이 보여야 한다. 가장 좋은 축은 관광객 쏠림 완화다.

추천 데이터:

- 지역별 관광객 방문 통계
- 관광지별 방문객 수
- 시간대/월별 방문 추이
- 카드 소비 또는 상권 매출 통계
- 읍면동별 유동인구
- 지역 행사/상권 활성화 데이터

제품 적용:

- “혼잡 예상 지역 대신 인접 권역 추천”
- “성산 쏠림을 구좌/표선으로 분산”
- 지역별 공공데이터 커버리지와 방문 분산 지표 결합
- 로컬 상권 소비 유도 경로 추천

대회 메시지:

> 관광객에게는 덜 붐비는 선택지를, 지역에는 소비 분산 효과를 제공합니다.

구현 우선순위: 높음

주의:

- 실시간 혼잡을 확보하지 못하면 “과거 방문 통계 기반 혼잡 가능성”으로 표현해야 한다.
- 혼잡도를 단정하지 말고, 데이터 기준과 기간을 UI에 표시해야 한다.

참고 출처:

- 제주데이터허브: https://www.jejudatahub.net/
- 공공데이터포털 문화관광 분류: https://www.data.go.kr/

### 4.4 숙박 데이터

이미 비짓제주 확장 카테고리로 구조화되어 있다. `accommodation` 카테고리가 `SEARCHABLE_CATEGORIES`에 포함되어 있고, `stay` 순간 카드가 `accommodation`으로 연결되었다.

따라서 숙박 데이터 자체를 새로 붙이는 과제가 아니라, 숙소를 “예약 추천”이 아닌 “동선 기준점”으로 쓰는 기능과 시연을 강화하는 것이 과제다. 예약 가능 여부와 가격은 실시간성이 강하므로 공공데이터만으로 단정하면 안 된다.

추천 데이터:

- 관광숙박업 인허가 데이터
- 숙박시설 위치/업종/주소
- 야영장/휴양펜션/관광호텔 등 유형 데이터
- 안전·위생·인허가 상태

제품 적용:

- 숙소를 “예약 추천”이 아니라 “동선 기준점”으로 사용
- 숙소 주변 30분 내 일정 조립
- 마지막 날 공항 이동 부담 점수 계산
- 부모님/아이 동반 시 숙소 주변 실내·병원·교통 접근성 확인

대회 메시지:

> 숙소를 판매하지 않고, 공공데이터 기반 여행 동선의 기준점으로 활용합니다.

현재 상태: 구조화 완료

남은 일:

- 숙소 주변 30분권 일정 조립 시연
- 공항 이동 부담과 숙소 위치를 연결
- 가격/예약 가능 여부는 말하지 않는 안전 문구 유지

주의:

- 가격, 예약 가능 여부, 리뷰 평점은 공공데이터 근거가 아니면 추천 근거로 쓰지 않는다.

### 4.5 축제·행사 데이터

이미 비짓제주 확장 카테고리로 구조화되어 있다. `festival` 카테고리가 있고, `festival_event` 순간 카드가 `festival`로 연결되었다. 시연력이 매우 좋기 때문에 신규 데이터 추가보다 날짜 매칭과 주변 동선 조립을 강화해야 한다.

추천 데이터:

- 축제/행사명
- 기간
- 장소
- 주최/주관
- 행사 좌표 또는 주소
- 행사 상태/변경 공지

제품 적용:

- 여행 날짜와 축제 기간 매칭
- 축제 주변 식사·카페·주차장 조립
- 날씨 위험이 있으면 실내 문화 대체안 제시
- 지역 분산형 축제 코스 제안

대회 메시지:

> 여행 날짜와 공공 행사 데이터를 대조해, 우연히 놓치는 제주 경험을 일정 안으로 끌어옵니다.

현재 상태: 구조화 완료

남은 일:

- 여행 시작일/종료일과 축제 기간 매칭
- 축제 주변 식사·카페·주차장 조립
- 날씨 위험 시 실내 문화 대체안 연결

### 4.6 특산물·전통시장·로컬 소비 데이터

부분적으로 구조화되어 있다. `shopping` 카테고리와 `souvenir_shopping` 순간 카드는 연결되었고, 전통시장·오일장은 기존 `market` 카드와도 연결된다. 다만 “특산물”을 별도 공공데이터로 깊게 구조화한 것은 아직 아니다.

2025년 수상작에도 AI 기반 제주 특산물 판매 솔루션이 있었으므로, 우리는 판매 솔루션이 아니라 “여행 동선 안의 로컬 소비 연결”로 차별화해야 한다.

추천 데이터:

- 전통시장/오일장 위치와 운영일
- 특산물 판매처
- 로컬푸드 직매장
- 착한가격업소
- 모범음식점/위생등급
- 가능하면 제주개발공사 또는 JDC 관련 지역 상생 데이터

제품 적용:

- 마지막 날 선물 구매 동선
- 오일장 날짜와 여행 기간 매칭
- 숙소/공항 이동 전 보냉 필요 특산물 안내
- 지역 상권 소비 유도 카드

대회 메시지:

> 여행 추천을 지역 소비로 연결해 제주 로컬 상권에 데이터 기반 방문 흐름을 만듭니다.

현재 상태: 쇼핑/시장 축은 구조화, 특산물 전용 데이터는 추가 후보

남은 일:

- 마지막 날 특산물·기념품 동선 시연
- 전통시장/오일장 날짜 매칭 강화
- 특산물 전용 데이터는 있으면 P1로 검토

주의:

- 판매 재고, 가격, 배송 가능 여부는 실시간 상거래 데이터가 없으면 단정하지 않는다.

### 4.7 문화·실내 대체지 데이터

이미 비짓제주 확장 카테고리로 구조화되어 있다. `culture` 카테고리와 `culture_stop` 순간 카드가 연결되었다. 날씨 리스크와 묶으면 매우 강력하다. 비, 폭염, 강풍이 있을 때 AI가 야외 일정을 줄이고 문화 공간으로 대체하는 시연이 가능하다.

추천 데이터:

- 박물관/미술관/전시관
- 공공문화시설
- 도서관/공연장
- 실내 관광지
- 휴관일 또는 운영시간 데이터

제품 적용:

- 우천/폭염/강풍 대체 일정
- 아이/부모님 동반 실내 코스
- 전시·문화 공간 주변 카페/식사 조립

대회 메시지:

> 날씨 변수를 공공데이터로 감지하고, AI가 실내 문화 대체 코스를 제시합니다.

현재 상태: 구조화 완료

남은 일:

- 날씨 리스크 발생 시 문화 카드 자동 대체 설명
- 휴관일/운영시간은 공공데이터에 없으면 단정하지 않는 문구 유지
- 부모님/아이 동반 실내 코스 시연

### 4.8 무장애·안전·편의시설 데이터

사회적 가치 점수에 직접 도움이 된다. “누구나 갈 수 있는 제주”는 발표 메시지가 강하다.

추천 데이터:

- 장애인 편의시설
- 공중화장실
- 유아 동반 편의시설
- 응급의료기관/약국
- 무더위쉼터
- 해수욕장 안전/개장 정보
- 재난안전 문자 또는 위험지역 데이터

제품 적용:

- 부모님/아이/교통약자 동행 시 경고 및 대체 후보
- 화장실/의료 접근성 배지
- 폭염·강풍·호우 시 야외 일정 경고
- 해변 개장기간과 안전 신호 확인

대회 메시지:

> 제주 여행을 더 많은 사람이 안전하게 선택할 수 있도록 공공 안전·편의 데이터를 AI가 읽어줍니다.

구현 우선순위: 중상

### 4.9 제주개발공사/JDC 관련 데이터

공고문상 각 기관별 공공데이터 활용 시 가점 2점이 있으므로, 주관기관 데이터를 연결할 수 있으면 발표 서사가 좋아진다.

추천 방향:

- JDC: 관광단지, 공공시설, 지역상생, 제주국제자유도시 관련 공개 데이터 확인
- 제주특별자치도개발공사: 삼다수, 지역상생, 친환경, 물류/자원순환 관련 공개 데이터 확인

제품 적용 아이디어:

- 물 보충/친환경 여행 준비 카드
- 지역상생 공간 또는 공공시설 안내
- JDC 관리 권역/시설과 관광 코스 연결

구현 우선순위: 중간

주의:

- 실제로 개방된 API/파일이 있는지 먼저 확인해야 한다.
- 억지로 붙이면 심사위원에게 “가점용 끼워넣기”로 보일 수 있으니, 시연 가치가 있는 데이터만 채택한다.

## 5. 최우선 데이터 조합

시간이 촉박하므로 전부 붙이면 안 된다. 우승 가능성 기준으로는 다음 5개 조합이 가장 좋다.

### 조합 A: 기상 리스크 + 실내 대체 코스

필요 데이터:

- 기상청 단기예보/초단기예보
- 비짓제주 문화/카페/쇼핑/숙박
- 주차장/버스정류장

시연:

> “내일 성산에 비와 바람이 있으면, 오름 대신 전시·카페·시장 중심으로 바꾸고, 주차/버스 접근성까지 보여준다.”

심사 효과:

- AI 활용도 높음
- 기상기후 특별상 대응
- 사용자 체감이 큼

### 조합 B: 공항 중심 마지막 날 코스

필요 데이터:

- 비짓제주 관광/음식/쇼핑
- 제주공항 위치/교통
- 주차장/버스정류장
- 날씨

시연:

> “오후 3시 비행기라면 공항 30~40분권 안에서 식사, 특산물 쇼핑, 짧은 산책만 조립한다.”

심사 효과:

- 실제 여행 문제 해결
- 짧고 명확한 시연 가능

### 조합 C: 부모님과 가는 접근성 코스

필요 데이터:

- 주차장
- 버스정류장
- 무장애/화장실/의료시설
- 날씨
- 관광지 좌표

시연:

> “부모님 동행이면 오름 난이도와 교통 부담을 낮추고, 주차·화장실·실내 대체지를 우선한다.”

심사 효과:

- 사회적 가치
- 기존 추천 앱과 차별화

### 조합 D: 축제 기간 로컬 소비 코스

필요 데이터:

- 축제/행사
- 전통시장/특산물
- 음식점 위생등급
- 지역별 방문/혼잡 통계

시연:

> “여행 기간과 맞는 축제를 찾고, 주변 로컬 식사·특산물 구매 동선으로 지역 소비를 연결한다.”

심사 효과:

- 사업화 가능성
- 지역 경제 기여

### 조합 E: 수정요청 기반 정보 신뢰 검증

필요 데이터:

- 콘텐츠 수정요청
- 비짓제주 장소 원본
- 웹 출처 검색

시연:

> “블로그에서 본 장소를 물으면, 공공데이터 수정요청 이력과 웹 출처를 대조해 그대로 믿어도 되는지 알려준다.”

심사 효과:

- AI 신뢰성
- 정보 불신 문제 해결
- 현재 프로젝트의 가장 강한 차별점

## 6. 개발 백로그

중요한 정정: 아래 백로그는 “데이터를 처음 붙이는 일”이 아니라, **이미 구조화된 데이터를 경진대회용 완성 기능으로 끌어올리는 일**이다.

### P0: 반드시 해야 하는 것

1. 기상청 데이터 기반 날씨 리스크 점수의 시연용 설명 강화
2. 야외/실내 순간 카드 자동 대체 규칙
3. 축제·숙박·쇼핑·문화 확장 카드의 데모 플로우 고정
4. 콘텐츠 수정요청 기반 신뢰 검증 대표 사례 선정
5. 사업계획서용 “이미 구현된 공공데이터 활용 현황” 표 정리
6. 하루방 답변에 “왜 이 데이터가 이 결정을 만들었는지” 설명 강화

### P1: 하면 우승력이 올라가는 것

1. 축제·행사 날짜 매칭 고도화
2. 숙박 위치를 기준점으로 하는 주변 일정 조립
3. 특산물·전통시장 마지막 날 동선
4. 버스 노선/도착 정보 등 교통 접근성 고도화
5. 지역 분산 추천 점수

### P2: 시간이 남으면 하는 것

1. JDC/제주개발공사 관련 공개 데이터 연결
2. 유동인구/혼잡 통계 기반 회피 추천
3. 무장애 관광 상세 정보
4. 전기차 충전소 기반 렌터카 여행 보조
5. 사업자용 지역 상권 리포트

## 8. 발표에서 보여줄 핵심 데모

### 데모 1: 날씨가 일정을 바꾸는 장면

입력:

> “부모님과 2박 3일, 성산과 구좌, 오름과 바다 산책을 가고 싶어요.”

보여줄 것:

- 기상청 예보에서 강풍/비 신호 감지
- 오름·바다 산책 위험도 표시
- 문화/카페/쇼핑 대체안 제안
- 주차장/정류장 접근성 확인

### 데모 2: 공항 근처 마지막 날

입력:

> “오후 비행기라 공항 근처에서 무리 없는 코스 짜줘.”

보여줄 것:

- 제주공항 위치를 웹/공공 출처로 확인
- 공항 이동 부담이 낮은 후보만 조립
- 특산물 쇼핑, 식사, 짧은 산책을 묶음

### 데모 3: 블로그 정보 검증

입력:

> “이 장소 아직 운영하는 거 맞아?”

보여줄 것:

- 비짓제주 원본
- 콘텐츠 수정요청 이력
- 웹 출처 보강
- 확인 가능/확인 불가/재확인 필요를 구분

## 9. 문서 작성 관점의 한 문장

사업계획서 첫 페이지에는 다음 문장을 중심에 둔다.

> 제주를 담다는 흩어진 제주 공공데이터를 AI가 여행자의 상황에 맞게 검증·조립하여, 날씨·교통·정보 변경으로 인한 여행 실패를 줄이고 지역 소비를 분산시키는 제주형 신뢰 여행 플랫폼입니다.
