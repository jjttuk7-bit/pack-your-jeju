# PDF 플랜 사용자 일정 추가 설계

## 목표

사용자가 PDF 플랜을 마지막으로 조합하는 단계에서 새로 떠오른 약속, 이동, 구매, 체크인 같은 개인 일정을 직접 추가할 수 있게 한다.

추가한 일정은 PDF 작업 초안에만 머물지 않고 `내 여행플랜`에도 즉시 반영한다. 사용자가 시간을 입력한 일정은 날씨·동선 재조합이 임의로 옮기지 않는 고정 일정으로 취급한다.

## 제품 원칙

- PDF 플랜은 최종 출력 전 사용자가 여행 일정을 완성하는 편집 공간이다.
- 사용자가 `일정 추가`를 확정하기 전에는 원본 플랜을 바꾸지 않는다.
- 확정한 일정은 PDF 작업 초안과 내 여행플랜에 같은 ID로 한 번씩 추가한다.
- 추가 직후에는 양쪽 반영을 함께 취소하는 되돌리기를 제공한다.
- 이후 PDF 카드의 일반 편집과 제외는 기존 PDF 작업 초안 정책을 따른다.
- 출처가 없는 개인 일정을 공식 정보처럼 표현하지 않고 `사용자 직접 입력`으로 표시한다.

## 입력 항목

- 일정명: 필수, 최대 80자
- Day: 필수, 폼을 연 Day를 기본값으로 사용
- 시간: 선택, `HH:mm`
- 장소·주소: 선택, 최대 200자
- 메모: 선택, 최대 800자

시간을 입력하면 `startTime`을 저장하고 `fixed: true`로 설정한다. 시간을 입력하지 않으면 `fixed: false`로 설정한다.

## 화면 흐름

### 일정 입력

- 각 Day 카드 하단에 `일정 직접 추가` 버튼을 둔다.
- 버튼을 누르면 해당 Day 안에서 인라인 입력 폼을 펼친다.
- Day는 현재 Day로 미리 선택하되 여행 기간 안의 다른 Day로 변경할 수 있다.
- 일정명이 비어 있거나 시간이 유효하지 않으면 추가 버튼을 비활성화하고 원인을 안내한다.
- 모바일에서는 필드를 한 열로 배치하고 입력 폼이 고정 하단 버튼에 가리지 않게 한다.
- 폼의 닫기 또는 Escape는 입력 중인 값만 폐기하며 PDF 편집기 자체는 닫지 않는다.

### 추가 결과

- 추가 성공 시 입력 폼을 닫고 대상 Day의 마지막 순서에 새 카드를 표시한다.
- 새 카드에는 `직접 추가` 배지를 표시한다.
- 시간이 있는 카드에는 `고정 일정` 상태를 함께 표시한다.
- 하단 임시저장 상태 근처에 `일정을 내 여행플랜에도 추가했어요 · 되돌리기`를 표시한다.
- 추가된 카드의 Day, 순서, PDF 메모는 기존 PDF 카드와 동일하게 편집할 수 있다.

### 제외와 되돌리기

- 추가 직후 제공하는 되돌리기는 같은 ID의 일정을 PDF 초안과 내 여행플랜에서 함께 제거한다.
- 되돌리기 이후 Day별 순서를 다시 매긴다.
- 추가가 확정된 뒤 카드의 `초안에서 제외`를 누르면 기존 정책대로 PDF 작업 초안에서만 제거하고 내 여행플랜에는 유지한다.

## 상태 구조와 데이터 흐름

`PlanPdfEditor`는 입력 폼과 검증을 담당한다. `PackingDashboard`는 PDF 작업 초안과 내 여행플랜을 연결하는 추가·되돌리기 동작을 조정한다.

```text
Day별 일정 입력
  -> 필드 검증
  -> 고유 ID를 가진 user_added TravelPlanItem 생성
  -> selectedPlanItems에 추가
  -> 같은 ID의 PlanPdfDraftItem을 대상 Day 마지막에 추가
  -> 작업 초안 갱신 시각과 최근 추가 ID 기록
  -> 성공 상태와 되돌리기 노출
```

생성하는 원본 일정은 다음 계약을 따른다.

```ts
interface PdfCustomScheduleInput {
  name: string;
  day: number;
  startTime: string | null;
  address: string | null;
  note: string | null;
}

const item: TravelPlanItem = {
  id,
  name,
  moment: 'user_added',
  source: 'user_added',
  day,
  startTime,
  fixed: Boolean(startTime),
  address,
  note,
};
```

PDF 초안 항목은 같은 ID와 일정 필드를 사용하고 `pdfMemo`를 원본 `note`로 초기화한다.

## 동기화 규칙

- PDF에서 새 일정 확정: PDF 초안과 내 여행플랜에 즉시 추가
- 추가 직후 되돌리기: 양쪽에서 함께 제거
- 추가 후 PDF Day·순서·메모 편집: PDF 작업 초안에서만 변경
- 추가 후 `초안에서 제외`: PDF 작업 초안에서만 제거
- 내 여행플랜에서 해당 일정 삭제: 기존 원본 삭제 동기화로 PDF 초안에서도 제거
- 하루방 재조합: 사용자 확인 후 다른 PDF 편집 내용과 함께 새 초안으로 교체

PDF 편집 단계의 Day·순서·메모 변경은 원본을 자동으로 계속 수정하지 않는다. 새 일정 확정 시 입력한 초기 값만 내 여행플랜에 반영하고, 그 뒤의 PDF 편집은 기존 작업 초안 분리 원칙을 유지한다.

## 예외 처리

- 일정명 앞뒤 공백을 제거한 결과가 비어 있으면 추가하지 않는다.
- Day가 여행 기간을 벗어나면 1일부터 마지막 Day 사이로 보정한다.
- 시간은 유효한 `HH:mm` 값만 허용한다.
- 중복 클릭과 오래된 편집 콜백이 동일 ID를 두 번 추가하지 못하게 한다.
- 내 여행플랜 반영이 완료되지 않으면 PDF 작업 초안에도 일정이 남지 않게 한다.
- 되돌릴 일정이 원본에서 먼저 삭제됐으면 PDF 초안에 남은 동일 ID만 안전하게 제거한다.
- PDF 생성 실패와 편집기 재열기 뒤에도 추가한 일정과 초안을 유지한다.

## 접근성

- `일정 직접 추가` 버튼의 접근 가능한 이름에 Day를 포함한다.
- 필수값과 시간 오류는 입력 필드와 연결해 안내한다.
- 추가 성공과 되돌리기 결과는 `aria-live="polite"`로 알린다.
- 입력 폼은 키보드만으로 열기, 작성, 취소, 확정할 수 있어야 한다.
- 입력 폼이 열린 동안 첫 Escape는 폼만 닫고 PDF 편집기는 유지한다.

## 검증

- 일정명 필수값과 각 필드 글자 수 제한을 검증한다.
- 입력 폼을 연 Day가 기본 선택되는지 검증한다.
- 선택한 Day 마지막 순서에 일정이 추가되는지 검증한다.
- 시간이 있으면 `fixed: true`, 없으면 `fixed: false`가 되는지 검증한다.
- PDF 초안과 내 여행플랜에 같은 ID가 한 번씩 추가되는지 검증한다.
- 추가 직후 되돌리기가 양쪽 일정을 함께 제거하는지 검증한다.
- 이후 `초안에서 제외`가 내 여행플랜을 변경하지 않는지 검증한다.
- 편집기를 닫았다 다시 열어도 추가한 일정이 유지되는지 검증한다.
- 일반 PDF 초안과 하루방 초안에서 같은 동작을 검증한다.
- 입력 폼 Escape 처리, 키보드 접근성, 모바일 배치를 검증한다.
- 기존 PDF 생성, 제외·복원, 새 원본 장소 승인, 하루방 초안 교체 테스트가 회귀하지 않는지 검증한다.
