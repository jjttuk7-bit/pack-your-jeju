# 여행 플랜 PDF 작업 초안 설계

## 목표

사용자가 PDF 편집기에서 일부 장소를 제외하고, 편집기를 닫아 후보를 더 살펴본 뒤, 기존 편집 내용을 잃지 않고 다시 이어서 다듬을 수 있게 한다.

이 기능은 PDF용 작업 초안만 변경한다. `내 여행플랜에 담은 것`의 원본 장소와 일정은 PDF 편집기의 제외 동작으로 변경하지 않는다.

## 제품 원칙

- PDF 편집은 원본 여행플랜과 분리된 작업 초안이다.
- 편집 내용은 별도 저장 버튼 없이 즉시 임시저장한다.
- 임시저장은 현재 여행팩 화면이 유지되는 동안만 존재한다.
- 새로 담은 장소는 자동으로 기존 초안에 섞지 않고 사용자가 확인해 추가한다.
- 원본 플랜을 변경하거나 하루방 재조합으로 초안을 교체할 때는 결과를 예측할 수 있게 안내한다.

## 화면 흐름

### 장소 제외와 복원

- 각 PDF 장소 카드 우측 상단에 `초안에서 제외` 버튼을 둔다.
- 제외 즉시 카드가 사라지고 하단에 `초안에서 제외했어요 · 되돌리기`를 표시한다.
- `되돌리기`는 가장 최근에 제외한 장소를 기존 Day와 순서로 복원한다.
- 제외는 PDF 작업 초안에만 적용하며 원본 여행플랜에는 영향을 주지 않는다.

### 편집기 나가기

- 하단 `취소` 버튼을 `나가서 장소 더 보기`로 바꾼다.
- 닫기 버튼과 Escape도 동일한 동작을 한다.
- 제목, Day, 순서, 메모, 제외 상태는 편집 즉시 대시보드 메모리에 저장한다.
- 하단에 `변경사항 임시저장됨` 상태를 표시한다.

### 새 장소 반영

- 편집기를 닫은 뒤 원본 플랜에 장소를 추가하면 기존 PDF 작업 초안은 유지한다.
- 다시 편집기를 열면 `새로 담은 장소 N곳이 있어요` 안내를 표시한다.
- 사용자가 `초안에 추가`를 누른 장소만 작업 초안의 적절한 Day 마지막 순서에 들어간다.
- PDF에서 제외했던 기존 장소는 새 장소로 오인해 다시 제안하지 않는다.

### 초안 교체

- 하루방을 다시 조합할 때 기존 작업 초안에 편집 내용이 있으면 교체 확인을 거친다.
- 승인하면 새 하루방 결과로 작업 초안을 교체한다.
- 거절하면 기존 작업 초안을 유지한다.

## 상태 모델

`PackingDashboard`가 일반 PDF와 하루방 PDF에 공통으로 사용할 메모리 상태를 소유한다.

```ts
interface PlanPdfWorkspace {
  draft: PlanPdfDraft;
  composition: HarubanPlanDraft | null;
  knownSourceItemIds: string[];
  excludedItems: PlanPdfDraftItem[];
  updatedAt: string;
}
```

- `draft`: 현재 PDF에 실제 포함되는 제목과 장소
- `composition`: 하루방 조합의 날씨·동선 근거. 일반 PDF 초안이면 `null`
- `knownSourceItemIds`: 초안을 만들거나 사용자가 추가 여부를 처리한 원본 장소 ID
- `excludedItems`: 되돌리기와 새 장소 오인 방지를 위한 제외 기록
- `updatedAt`: 임시저장 상태 표시 시각

브라우저 `localStorage`에는 저장하지 않는다. 여행팩 대시보드가 언마운트되거나 여행을 초기화하면 작업 초안도 사라진다.

## 원본 플랜 동기화

- PDF에서 장소 제외: 작업 초안에서만 제거
- 원본에 장소 추가: 보류 목록으로 표시하고 사용자 승인 후 추가
- 원본에서 장소 삭제: PDF 작업 초안에서도 자동 제거
- 원본 일정 변경: 이미 작업 초안에서 편집한 Day·순서·메모는 유지
- 하루방 다시 조합: 사용자 확인 후 전체 작업 초안 교체

## 예외 처리

- 작업 초안이 0곳이면 `PDF 만들기`를 비활성화하고 장소 추가 안내를 표시한다.
- 새 장소가 이미 작업 초안에 있거나 제외 기록에 있으면 중복 추가하지 않는다.
- 원본에서 삭제된 장소가 최근 제외 기록에 있으면 되돌리기 대상에서도 제거한다.
- PDF 생성 실패 시 작업 초안은 유지하고 기존 오류 메시지를 표시한다.

## 접근성

- `초안에서 제외` 버튼에는 장소명을 포함한 접근 가능한 이름을 제공한다.
- 제외 결과와 임시저장 상태는 `aria-live="polite"`로 알린다.
- 되돌리기, 새 장소 추가, 초안 교체 확인은 키보드만으로 사용할 수 있어야 한다.
- Escape와 닫기 버튼은 생성 중이 아닐 때만 동작한다.

## 검증

- 일반 PDF 초안과 하루방 초안 각각에서 제외와 되돌리기를 검증한다.
- 닫기 버튼, Escape, `나가서 장소 더 보기` 후 재열었을 때 편집 상태가 복원되는지 검증한다.
- 새 원본 장소가 자동 추가되지 않고 승인 후 추가되는지 검증한다.
- 원본에서 삭제한 장소가 작업 초안에서도 제거되는지 검증한다.
- PDF 초안 편집이 `selectedPlanItems` 변경 콜백을 호출하지 않는지 검증한다.
- 빈 초안의 PDF 생성 차단과 기존 PDF 생성 회귀를 검증한다.
