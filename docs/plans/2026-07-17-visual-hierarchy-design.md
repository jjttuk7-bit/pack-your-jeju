# 시각 위계 1단계 설계

## 목표

기능과 화면 구조를 바꾸지 않고 랜딩페이지와 대시보드의 제목 줄바꿈, 작은 글자 가독성, 색상 역할, CTA 위계를 정리한다.

## 방향

기존의 제주 로컬·에디토리얼 감성을 유지한다. 새로운 테마를 만들지 않고 현재 디자인 토큰을 확장해 다음 역할을 일관되게 적용한다.

- 감귤 주황: 사용자가 지금 수행할 핵심 행동
- 짙은 초록: 탐색 및 중립적인 보조 행동
- 민트: 확인 완료 또는 신뢰 상태
- 황색: 확인이 필요하거나 주의가 필요한 상태
- 현무암: 제목과 본문

## 랜딩페이지

- 첫 화면과 마지막 전환점의 CTA만 채움형으로 강조한다.
- 중간 CTA는 외곽선 또는 밑줄형으로 낮춰 페이지의 행동 우선순위를 명확히 한다.
- `검색은 덜 하고, 제주는 더 오래 기억하세요.`는 단어 중간에서 끊기지 않도록 제목 폭과 반응형 크기를 조정한다.
- 보조문구는 모바일과 데스크톱 모두에서 12px 미만으로 내려가지 않게 하고 배경 대비를 높인다.

## 대시보드

- 기존 감귤색 CTA는 유지하되 경고와 확인 상태에는 각각 황색과 민트를 사용한다.
- 설명, 상태, 메타데이터의 최소 글자 크기와 명암 대비를 높인다.
- 기존 레이아웃과 상호작용 계약은 유지한다.

## 접근성 및 검증

- 주요 버튼의 포커스 표시와 최소 높이를 유지한다.
- 텍스트 색상은 일반 본문 기준 WCAG AA 대비를 목표로 한다.
- 컴포넌트 테스트에서 CTA 역할과 제목 줄바꿈 계약을 검증한다.
- 데스크톱 1440px와 모바일 390px에서 수평 넘침, 단어 중간 줄바꿈, CTA 위계를 시각 확인한다.

## 제외 범위

- 랜딩 섹션 높이와 전체 여백 압축
- 대시보드 카드 구조 변경
- 하루방 패널 위치 및 크기 변경
- 새로운 기능 또는 데이터 흐름 추가
