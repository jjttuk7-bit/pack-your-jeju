# 모바일 지도 안내문 배치 설계

## 문제

스마트폰 너비에서 제주 행정구역 지도 내부의 안내문이 지도 위에 절대 배치되어 있다. 안내문이 하단 지역 면을 시각적으로 가리고 포인터 이벤트도 받아, 사용자가 해당 영역의 지도를 누르기 어렵다.

## 선택한 설계

- 모바일에서는 안내문을 지도 그림 아래의 일반 문서 흐름에 배치한다.
- `sm` 이상 화면에서는 현재처럼 지도 우측 하단에 오버레이한다.
- 안내 문구, 지도 선택 로직, 행정구역 SVG와 데스크톱 시각 구성은 유지한다.
- 모바일 안내문은 지도 카드와 같은 테두리·배경 계열을 사용해 기존 디자인 언어를 유지한다.

## 검토한 대안

1. 오버레이에 `pointer-events-none`만 적용: 터치는 전달되지만 지도 일부가 계속 가려져 사용성이 충분히 개선되지 않는다.
2. 모바일에서 안내문 숨김: 터치 문제는 사라지지만 지도 사용법을 처음 접하는 사용자에게 필요한 설명도 사라진다.
3. 모바일에서 지도 아래로 이동: 지도 전체가 보이고 안내도 보존되어 가장 균형이 좋다. 이 방식을 채택한다.

## 검증

- 소스 회귀 테스트로 모바일 안내문이 일반 흐름이고 데스크톱 안내문만 절대 배치되는지 확인한다.
- TypeScript 검사와 프로덕션 빌드를 실행한다.
- 스마트폰 너비에서 지도 위 안내문이 사라지고 행정구역을 클릭할 수 있는지 브라우저로 확인한다.
