# 랜딩 공공데이터 선순환 밴드 디자인

## 목표

‘제주를 담다’가 공공데이터를 단순히 소비하는 여행 서비스가 아니라, 실제 여행에서 확인한 정보와 방문 경험을 다음 검증과 데이터 개선에 연결하는 서비스임을 일반 사용자도 쉽게 이해하게 한다.

핵심 문장은 다음과 같다.

> 여행으로 확인하고, 다시 강화하는 제주 공공데이터

## 배치

`LandingPage`의 어두운 초록색 ‘여행이 좋아지는 흐름’ 섹션 안에 배치한다.

- 네 개의 여행 단계 바로 아래에 둔다.
- 현재 하단의 `웹 출처 + 공공데이터 + 실제 방문 경험 → 나의 제주 플랜` 정보 바를 확장한다.
- 다음 흰색 ‘여행을 만드는 가장 쉬운 방법’ 섹션으로 넘어가기 전에, 여행 경험이 다시 다음 여행의 근거로 돌아간다는 결론을 제공한다.
- 별도 대형 섹션을 추가하지 않아 현재 랜딩의 길이와 흐름을 유지한다.

## 콘텐츠

### 제목

`여행으로 확인하고, 다시 강화하는 제주 공공데이터`

### 설명

`출발 전에는 공공데이터와 최신 웹 정보를 확인하고, 여행 후에는 달라진 정보와 실제 경험을 기록해 다음 검증과 데이터 개선에 연결합니다.`

### 선순환 단계

1. `공공데이터로 확인`
   - 장소와 기본 정보를 공공데이터 및 출처로 확인한다.
2. `실제 여행`
   - 사용자가 선택한 후보를 자신의 제주 여행에서 경험한다.
3. `방문 기록·수정 신호`
   - 달라진 정보, 방문 여부, 만족도와 수정이 필요한 내용을 기록한다.
4. `다음 여행의 더 나은 근거`
   - 기록을 다음 검증과 데이터 개선에 연결한다.

## 정보 구조

기존 정보 바를 두 층으로 구성한다.

1. 상단에는 제목과 설명을 배치한다.
2. 하단에는 네 단계의 선순환 흐름을 배치한다.
3. 기존의 `나만의 제주를 담아보기` 버튼은 밴드 우측 또는 하단에 유지한다.
4. 새로운 CTA는 추가하지 않는다.

사용자는 먼저 선순환의 의미를 읽고, 기존 CTA로 자연스럽게 여행 만들기를 시작한다.

## 시각 디자인

- 기존 어두운 초록 배경과 주황색 강조색을 유지한다.
- 밴드는 현재 섹션과 분리되는 얇은 상·하단 선 또는 반투명 패널로 표현한다.
- 제목은 흰색 세리프 계열로, 설명은 낮은 대비의 흰색 보조 문장으로 표시한다.
- 각 단계는 아이콘, 짧은 제목, 연결선을 함께 사용한다.
- 연결선과 화살표는 장식 요소이며, 단계 순서는 숫자와 텍스트로도 식별할 수 있어야 한다.
- 선순환을 강조하되 원형 다이어그램처럼 복잡하게 만들지 않고, 현재 네 단계 인포그래픽의 시각 언어를 이어간다.

## 반응형

### 데스크톱

- 네 단계를 한 줄로 배치한다.
- 단계 사이를 얇은 선과 화살표로 연결한다.
- 제목·설명과 기존 CTA는 상단 또는 양 끝에 균형 있게 배치한다.

### 모바일

- 제목과 설명을 먼저 보여준다.
- 네 단계를 세로로 쌓는다.
- 연결선은 왼쪽 세로축으로 전환한다.
- 가로 스크롤을 만들지 않는다.

## 표현 원칙

- 사용자의 기록이 공공 원천 데이터를 자동으로 수정한다고 표현하지 않는다.
- `다음 검증과 데이터 개선에 연결한다`는 실제 서비스 범위의 표현을 사용한다.
- 공공데이터만으로 모든 여행 정보가 완성된다고 주장하지 않는다.
- 웹 출처, 공공데이터, 실제 방문 경험의 역할을 구분한다.

## 접근성

- 각 단계는 아이콘 없이도 이해되는 텍스트 제목을 가진다.
- 색상만으로 단계와 방향을 구분하지 않는다.
- 장식용 화살표와 연결선은 스크린 리더에서 제외한다.
- 본문과 보조 문구는 어두운 배경에서 충분한 대비를 확보한다.
- 모션을 추가할 경우 `prefers-reduced-motion` 환경에서는 정적인 상태로 표시한다.

## 테스트

- 랜딩에 핵심 제목과 네 단계 문구가 모두 렌더링되는지 확인한다.
- 기존 `나만의 제주를 담아보기` CTA가 유지되는지 확인한다.
- 새로운 주요 CTA가 추가되지 않았는지 기존 시각 역할 테스트로 검증한다.
- 모바일 폭에서 단계가 세로 배치되고 문서 가로 넘침이 없는지 브라우저로 확인한다.
- 데스크톱에서 기존 네 단계 인포그래픽과 새 밴드 사이의 간격과 위계가 자연스러운지 시각 검토한다.

## 제외 범위

- 방문 기록을 서버에 저장하는 새 기능
- 공공데이터 원천을 자동 수정하는 기능
- 새로운 API 또는 데이터베이스 변경
- 별도 공공데이터 소개 페이지
- 새로운 랜딩 CTA
