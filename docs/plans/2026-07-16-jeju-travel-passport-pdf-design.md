# 제주 여행 패스포트 PDF 설계

## 목표

현재 `내 플랜 저장`은 사용자가 담은 장소와 메모를 텍스트 파일로 저장한다. 이를 여행 출발 전 설렘을 주면서 현장에서도 실제로 사용할 수 있는 `제주 여행 패스포트`형 PDF로 전환한다.

PDF의 중심은 공공데이터가 자동 추천한 전체 후보가 아니라 사용자가 실제로 `내 여행플랜`에 담은 장소다. 하루방 웹 추천, 공공데이터 후보, 사용자가 직접 추가한 장소를 하나의 여정으로 편집하고 다운로드할 수 있어야 한다.

## 검토한 접근

### 1. 여행 패스포트형 — 채택

제주 입도 티켓과 여권을 연상시키는 표지, Day별 여행 티켓 카드, 제주 스탬프와 풍경 그래픽을 사용한다. 감성적인 결과물이지만 주소, 지도, 예약·운영 재확인, 준비물과 출처 페이지를 포함해 여행 중에도 사용할 수 있다.

### 2. 일반 일정표형

표와 시간표 중심이라 정보 탐색은 빠르지만 현재 서비스의 따뜻한 제주 브랜드와 여행 시작의 감정을 충분히 살리지 못한다.

### 3. 여행 스크랩북형

기념품과 기록물로는 매력적이지만 여행 전 일정 확인과 현장 사용성이 떨어지고, 이미지 의존도가 높아진다.

## 사용자 흐름

1. 사용자가 공공데이터 후보, 하루방 웹 추천, 직접 찾은 장소를 `내 여행플랜`에 담는다.
2. `여행 플랜 PDF` 버튼을 누르면 다운로드 전 편집 창이 열린다.
3. 시스템은 여행 기간과 현재 장소 순서를 기준으로 장소를 Day별로 자동 배치한다.
4. 사용자는 다음 항목을 수정한다.
   - PDF 여행 제목
   - 각 장소의 Day
   - 같은 Day 안의 방문 순서
   - 장소별 짧은 메모
5. `PDF 만들기`를 누르면 서버가 결정론적으로 PDF를 생성한다.
6. 브라우저가 의미 있는 파일명으로 PDF를 다운로드한다.

## 편집 방식

혼합형을 사용한다.

- 첫 진입 시 자동 배치한다.
- 기본 자동 배치는 사용자가 담은 순서를 보존하면서 전체 여행 일수에 고르게 나눈다.
- 장소에 Day 정보가 이미 있으면 해당 값을 우선한다.
- 사용자는 드래그 앤 드롭 대신 접근성이 단순한 `Day 선택`, `위로`, `아래로` 버튼으로 순서를 수정한다.
- 편집 결과는 PDF 요청에만 사용하며, 1차 버전에서는 원본 `selectedPlanItems`의 저장 구조를 변경하지 않는다.
- 추후 일정 편집 저장 기능을 추가할 수 있도록 요청 계약에는 `day`, `order`, `memo`를 명시한다.

## PDF 정보 구조

### 1. 표지 — 제주 여행 패스포트

- 사용자 지정 제목, 기본값 `나의 제주 여행`
- 여행 기간, 선택 지역, 동행자, 여행 목적
- 담은 장소 수
- 감귤 여권 도장, 제주 바다·오름·돌담을 추상화한 벡터 그래픽
- 표지 문구: `근거를 확인하고, 나만의 제주를 출발합니다.`

외부 장소 사진은 사용하지 않는다. 저작권, 깨진 링크, 서버 다운로드 지연을 피하고 서비스 자체 그래픽으로 일관성을 유지한다.

### 2. 여행 한눈에 보기

- Day별 장소 수와 주요 지역
- 출처별 장소 수
  - 하루방 웹검색 추천
  - 공공데이터 후보
  - 사용자 직접 추가
- 여행 전 최종 확인 안내
- 선택한 순간과 여행 목적 요약

### 3. Day별 여행 여정

각 장소를 제주 탑승권 또는 여행 티켓 형태의 카드로 표현한다.

- Day와 방문 순서
- 장소명
- 주소
- 장소 출처 유형
- 선택 이유 또는 사용자 메모
- `확인됨`, `방문 전 재확인`, `직접 추가·미검증` 상태
- 예약, 운영시간, 날씨, 주차 등 `check_required` 항목
- 지도 검색 QR 코드
- 여행 중 손으로 적을 수 있는 짧은 메모 줄

정확한 방문 시각과 이동시간은 근거 없이 자동 생성하지 않는다. 1차 버전은 순서 중심의 여정으로 표시한다.

### 4. 나의 제주 준비물

- 현재 선택 순간과 장소에 따라 이미 계산된 맞춤 준비물
- 출발 전 체크리스트
  - 운영시간·휴무 확인
  - 예약 확인
  - 날씨 확인
  - 이동·주차 확인
- 사용자가 직접 추가한 준비 메모가 있으면 포함한다.

### 5. 근거 확인 페이지

본문은 여행답게 간결하게 유지하고 상세 근거는 마지막에 모은다.

- 장소명
- 출처명과 URL
- 확인 날짜
- `공공데이터`, `하루방 웹검색`, `사용자 직접 추가` 구분
- 방문 전 재확인 대상
- 정보는 변경될 수 있다는 안내

URL은 클릭 가능한 링크로 만들고, 본문 장소 카드의 지도 QR과 구분한다.

## 상태 표현

| 입력 상태 | PDF 표시 |
|---|---|
| 공공데이터, verified | 확인됨 |
| 공공데이터, caution 또는 contradicted | 확인 필요 |
| 하루방 웹검색 추천 | 방문 전 재확인 |
| 사용자 직접 추가 | 직접 추가·미검증 |

강한 단정 대신 사용자가 실제로 해야 할 다음 행동을 보여준다.

## 데이터 계약

기존 `/pack/pdf`는 공공데이터 팩을 다시 조립하는 별도 기능으로 유지한다. 개인 플랜 전용 `POST /plan/pdf`를 추가한다.

요청 예시:

```json
{
  "title": "혼자 떠나는 제주시 힐링 여행",
  "travel": {
    "regions": ["jeju_city"],
    "start_date": "2026-07-16",
    "days": 2,
    "companion": "solo",
    "purpose": "healing",
    "moments": ["local_food"]
  },
  "items": [
    {
      "id": "web-example",
      "name": "우진해장국",
      "day": 1,
      "order": 1,
      "source": "web_search",
      "address": "제주시 서사로 11",
      "memo": "공항 도착 후 아침",
      "source_title": "Visit Jeju",
      "source_url": "https://example.com",
      "checked_at": "2026-07-16T00:00:00+09:00",
      "check_required": ["operating", "parking"]
    }
  ],
  "packing_items": ["보조배터리", "얇은 바람막이"]
}
```

서버는 장소 사실을 추가하거나 보완하지 않는다. 전달받은 사용자 플랜 정보와 기존 준비물만 조립한다.

## 프론트 구성

### `PlanPdfEditor`

- 여행 제목 입력
- Day별 장소 그룹
- 장소별 Day 선택
- 같은 Day 내 위·아래 이동
- 장소별 PDF 메모
- 빈 플랜 방지 안내
- PDF 생성 진행·실패 상태

### 대시보드 버튼

- 기존 `내 플랜 저장`을 `여행 플랜 PDF`로 변경한다.
- 플랜이 비어 있으면 비활성화하고 먼저 장소를 담도록 안내한다.
- 텍스트 공유 기능은 유지한다.

## 오류 처리

- 선택 장소가 없으면 API를 호출하지 않는다.
- 잘못된 Day, 중복 order, 너무 긴 제목·메모는 서버에서 정규화하거나 400으로 반환한다.
- PDF 생성 실패 시 기존 플랜은 유지하고 다운로드 창에서 재시도할 수 있다.
- 한글 폰트가 없으면 기존 CID 폰트 fallback을 사용한다.
- 지도 QR 생성 실패 시 주소 텍스트만 표시하고 PDF 생성은 계속한다.

## 테스트와 시각 검증

- 자동 Day 배치와 순서 이동 단위 테스트
- 프론트 요청에 선택 장소·메모·Day·order가 포함되는 계약 테스트
- API 스키마 검증 테스트
- 웹검색·공공데이터·직접 추가 장소가 PDF에 모두 나타나는 생성 테스트
- PDF 텍스트 추출로 제목, 장소명, 출처 페이지 확인
- PDF 전 페이지를 PNG로 렌더해 겹침, 잘림, 한글 깨짐, 빈 페이지를 시각 검사
- 모바일과 데스크톱에서 편집 창 조작 검증

## 성공 기준

- 사용자가 담은 장소만 여행 여정에 포함된다.
- 다운로드 전에 Day, 순서, 제목, 메모를 수정할 수 있다.
- PDF가 여행 패스포트처럼 감성적이면서 현장에서 주소·확인 사항·지도를 사용할 수 있다.
- 출처와 불확실성을 숨기지 않으면서 본문은 여행 일정 중심으로 읽힌다.
- 외부 이미지 다운로드와 추가 LLM 호출 없이 안정적으로 생성된다.
