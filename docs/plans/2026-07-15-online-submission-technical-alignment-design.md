# 온라인 접수 입력문안 기술 중심 정합화 설계

## 목적

온라인 접수 화면의 제목·주요내용·대표이미지·첨부파일 안내를 최신 10페이지 사업계획서와 일치시킨다. 심사위원이 짧은 입력문만 읽어도 서비스의 실제 구현 기술, 공공데이터 활용 구조, 현재 MVP 수준, 개인 창업자의 실행 역량과 확장 가능성을 파악할 수 있어야 한다.

## 확정 방향

- 제목은 기억성과 핵심 메시지가 검증된 `여행으로 검증하고 다시 강화하는 제주 공공데이터, ‘제주를 담다’`를 유지한다.
- 주요내용은 기술을 전면에 두되 기술 용어만 나열하지 않고, 각 기술이 어떤 판단과 신뢰 문제를 해결하는지 연결한다.
- 공공데이터 라이프사이클을 전체 모델로, 여행 플랜을 그 라이프사이클의 활용·검증 접점으로 설명한다.
- 신청 주체는 팀이 아닌 개인이다. `대표자`를 쓰지 않고 `개인 창업자`, `신청자`, `개발자`를 사용한다.
- 현재 구현과 향후 고도화를 명확히 구분해 과장 없이 확장성을 보여준다.

## 문서 구조

### 1페이지: 복사할 제목과 주요내용의 도입부

1. 제목 입력값
2. 주요내용 복사 안내
3. 제품·서비스 정의
4. 핵심 기술·기능
5. 공공데이터 활용 규모와 연결 방식

도입부에는 다음 실제 기술을 심사 언어로 압축한다.

- Python 기반 ETL·정규화·체크섬 변경 감지
- PostgreSQL 관계형 검색, Full Text Search, GIN, `pg_trgm`
- 공영주차장 1km·버스정류장 500m 공간 연결
- 기상청 예보 정규화
- LLM Function Calling·RAG 기반 하루방 리서치
- 규칙 기반 Trust Engine과 점수 근거 표시
- React/Vite PWA, FastAPI, Pydantic, SQLAlchemy, PostgreSQL

### 2페이지: 라이프사이클·실행력·사업화

1. 세 가지 플랜 생성 경로
   - 공공데이터 후보
   - 하루방 웹검색 추천
   - 사용자 직접입력
2. 실제 여행과 피드백의 신뢰 축적
   - `PlanItem`, `Evidence`, `VisitFeedback`, `ModerationCase`, `PublicDataCorrection`
   - 중복 제출 방지 키, 원본 보존, 검토 후 버전형 보정
3. 현재 MVP와 검증 결과
   - 신뢰 엔진 골든셋 12/12
   - 백엔드 자동 테스트 229건
4. 개인 창업자 실행 역량
   - AI 엔지니어링 정규 교육과정 전 과정 이수, 공식 수료 절차 예정
   - Python 데이터 처리, ML, LLM·RAG
   - 60개 이상의 웹 서비스·프로토타입 저장소
5. 단계적 확장 및 B2C→B2B→B2G 사업화
6. 배포 서비스 URL

### 3페이지: 제출 보조 안내

1. 온라인 편집기 권장 서식
2. 대표이미지 규격과 권장 문구
3. 최신 사업계획서 첨부파일명
4. 저장 전 체크리스트

## 표현 원칙

- 핵심 문장은 짧게 쓰고, 기술명 뒤에는 사용자 가치 또는 데이터 신뢰 효과를 붙인다.
- `DB`, `RAG`, `Function Calling` 같은 용어는 실제 구현 설명에만 쓰며 일반적인 AI 홍보 문구로 확장하지 않는다.
- `대표자`, 팀 역량, 자동 원본 수정, 특정 모델 고정 표현은 사용하지 않는다.
- 서비스 URL은 DOCX와 PDF에서 클릭 가능하게 유지한다.

## 산출물

- `05_온라인접수_제목_주요내용_입력문안_최신본.docx`
- `05_온라인접수_제목_주요내용_입력문안_최신본.pdf`

두 파일은 `C:\Users\USER\Desktop\제주를담다 공모전 자료`에 저장한다. 저장소에는 재생성 가능한 스크립트, 검증기, 설계·구현 계획을 보존한다.

## 검증 기준

- PDF가 정확히 3페이지인지 확인한다.
- 제목, 서비스 URL, 최신 사업계획서 파일명이 정확한지 확인한다.
- 필수 기술·수치·개인 창업자 표현이 포함됐는지 자동 검사한다.
- 금지 표현과 이전 파일명이 없는지 검사한다.
- DOCX 하이퍼링크와 PDF URI 주석을 검사한다.
- 모든 페이지를 이미지로 렌더링해 잘림, 겹침, 지나친 밀도, 깨진 한글을 육안 확인한다.
- 심사위원 관점 질문으로 제품 정의, 기술, 공공데이터 활용, 신뢰 축적, MVP, 개인 역량, 사업화가 문서만으로 답변되는지 점검한다.
