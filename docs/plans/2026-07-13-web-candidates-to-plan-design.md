# Web Candidates To Plan Design

## Goal

하루방 웹검색을 읽기 전용 답변에서 선택 가능한 여행 후보로 전환하고, 사용자가 고른 장소를 기존 플랜과 방문 피드백 흐름에 연결한다.

## Data Flow

1. Responses API 웹검색이 출처가 포함된 Markdown 답변을 반환한다.
2. 백엔드는 굵게 표시된 고유 장소와 해당 구간의 원문 URL만 `place_candidates`로 구조화한다.
3. 프론트는 장소별 체크박스와 원문 링크를 표시한다.
4. 선택 장소는 `source=web_search`인 `TravelPlanItem`으로 로컬 여행 상태에 저장한다.
5. 일정, 짐 목록, 공유 문구와 방문 후 피드백은 기존 플랜 흐름을 재사용한다.

## Trust Boundaries

- 웹검색 후보는 공공데이터 후보로 표시하지 않는다.
- 검색 질문, 원문 URL, 출처명, 확인시각을 플랜 항목에 보존한다.
- 사용자 방문 피드백은 공공데이터 원본을 직접 수정하지 않고 별도 방문 신호로 취급한다.
- 출처 URL이 없는 장소는 선택 후보로 만들지 않는다.

## Interface

- 답변 아래에 `검색한 장소를 플랜에 담기` 영역을 표시한다.
- 여러 장소를 체크한 뒤 한 번에 플랜에 담을 수 있다.
- 이미 담긴 장소는 체크된 상태로 잠그고 중복 추가하지 않는다.
- 플랜에서는 `하루방 웹검색` 배지와 검색 원문 링크를 표시한다.
