# 하루방 무료 주장 근거 설계

## 목표

웹 출처에 입장료나 요금 안내가 없다는 이유만으로 장소가 무료라고 단정하지 않는다. 공식 또는 신뢰 가능한 운영 출처가 무료라고 명시한 경우에만 무료를 확정한다.

## 원인

현재 웹 검색 프롬프트에는 가격을 직접 확인한 경우에만 단정하라는 일반 규칙이 있지만, `요금 정보 부재 ≠ 무료` 규칙이 명시돼 있지 않다. 웹 검색과 최종 모델 답변이 일반 Markdown을 주고받기 때문에 “입장료 언급 없음”을 “무료 방문 가능”으로 확대하는 추론을 결정론적으로 차단하지 못한다.

## 선택한 방식

프롬프트 규칙과 최종 답변 검증을 함께 적용한다.

1. 웹 검색 프롬프트와 하루방 기본 시스템 프롬프트에 다음 규칙을 추가한다.
   - 요금·입장료 안내를 찾지 못한 것은 무료의 증거가 아니다.
   - 무료라는 명시는 공식 운영 주체 또는 신뢰 가능한 운영 출처가 직접 밝힌 경우에만 사용한다.
   - 근거가 정보 부재뿐이면 `별도의 입장료 안내는 확인되지 않았다`고 표현한다.
2. 최종 웹 답변에서 강한 무료 주장과 정보 부재 근거가 함께 나타나면 보수적인 문구로 완화한다.
3. 공식 무료 명시가 답변에 연결된 경우는 무료 표현을 유지한다.

## 결정론적 검증

웹 기반 최종 답변에 다음 두 조건이 동시에 있을 때만 완화한다.

- 강한 무료 주장: `무료`, `입장료 없이`, `요금 없이`, `자유롭게 방문 가능`
- 정보 부재 근거: `입장료 언급 없음`, `요금 안내 없음`, `가격 정보 없음`, `별도 안내를 찾지 못함`

완화 문구:

> 별도의 입장료 안내는 확인되지 않았습니다. 주차·체험·대여 시설을 이용하면 별도 비용이 생길 수 있어 방문 전 다시 확인해 주세요.

답변에 `공식 출처에서 무료라고 명시`, `입장료 무료로 명시`, `무료 개방을 공식 안내`처럼 직접 명시 근거가 있으면 완화하지 않는다.

## 적용 위치

- 웹 검색 결과 생성 직후
- 사전 검색 결과를 사용한 최종 LLM 답변 반환 직전
- 모델 장애 시 검색 결과 기반 폴백 반환 직전

이를 공통 함수로 처리해 경로별 누락을 방지한다.

## 테스트

- 정보 부재를 근거로 한 무료 문장을 완화한다.
- 공식 무료 명시가 있는 문장은 유지한다.
- 무료 또는 정보 부재와 무관한 답변은 변경하지 않는다.
- 웹 검색 프롬프트와 기본 시스템 프롬프트에 규칙이 포함된다.
- 하루방 전체 테스트, API 전체 테스트, 웹 전체 테스트, 타입 검사를 실행한다.
