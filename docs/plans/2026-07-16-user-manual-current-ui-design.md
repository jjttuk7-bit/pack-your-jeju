# 사용자 매뉴얼 최신 UI 개정 설계

## 목적

제출용 사용자 매뉴얼을 현재 배포 서비스와 일치시키면서 12페이지 구성을 유지한다. 초기 지도·후보 화면을 최신 화면으로 교체하고, 순간별 장소 대표이미지·지도 마커·최신 하루방 기능을 실제 구현 화면으로 증명한다.

## 확정 범위

### 표지

- 상단 서비스명 `제주를 담다`와 부제는 유지한다.
- `사용자 매뉴얼` 제목 뒤에 흐릿하게 보이는 대형 배경 글자 `담다`만 제거한다.
- 서비스 URL, QR 코드, PWA 안내 버전 표시는 유지한다.

### 화면 자산

기존 2026년 7월 14일 초기 캡처 4장을 현재 서비스 기준으로 교체한다.

1. 최신 랜딩 화면
2. 최신 제주 지도와 지역·여행 조건·순간 선택 화면
3. 제주팩 결과의 전체 요약·지도 마커·순간별 후보·체크리스트 화면
4. 장소 대표이미지가 보이는 순간별 후보 카드와 추가 후보 탐색 화면
5. 담은 장소·지도·플랜·체크리스트 연결 화면
6. 최신 하루방 패널의 웹검색 근거·출처·주의점·플랜 담기 화면

장소 카드는 이미지가 실제로 표시되는 후보를 사용한다. 화면 캡처에는 개인정보, API 키, 관리자 주소, 브라우저 북마크를 포함하지 않는다.

## 12페이지 구성

1. 표지
2. 빠른 시작
3. 앱처럼 설치하기(PWA)
4. 첫 화면에서 시작하기
5. 지역·여행 조건·순간 고르기
6. 제주팩 결과 화면 읽기
7. 후보 카드와 신뢰 표시 이해하기
8. 다른 후보 더 보기
9. 내 여행플랜 만들기·저장·공유
10. 하루방에게 조사 부탁하기
11. 방문 후 피드백 남기기
12. 모바일 사용 팁·문제 해결

페이지를 늘리지 않고 설명을 압축해 새 이미지를 배치한다.

## 페이지별 변경

### 5페이지

초기 지도 캡처를 현재 `TrustMapDashboard` 기반 제주 지도와 현재 순간 카드가 함께 보이는 화면으로 교체한다. 지역 선택 근거와 순간 선택 결과가 연결된다는 점을 설명한다.

### 6페이지

현재 제주팩 결과의 지도 마커, 순간별 후보 카드, 장소 대표이미지, 준비물 체크리스트를 한 화면 또는 두 개의 연결된 크롭으로 보여준다. 기존의 빈 지도·이미지 없는 후보 화면은 사용하지 않는다.

### 7~8페이지

실제 장소 이미지, 신뢰 배지, 출처 역할, 확인 필요 항목, `다른 후보 5곳 보기`와 전체 후보 수 표시를 최신 UI로 설명한다.

### 9페이지

플랜에 담은 장소와 지도 마커, 저장·공유, 직접 추가 메모가 현재 UI에서 어떻게 연결되는지 보여준다.

### 10페이지

기존 텍스트만 있는 하루방 안내를 최신 화면 중심으로 개정한다.

- 지역·일정·동행자·취향의 멀티턴 문맥 유지
- 공식·플랫폼·경험 출처를 조사한 근거형 답변
- 최신성·충돌·주의점 표시
- 하루방 후보를 내 여행플랜에 담기
- 리뷰 검증 진입

실제 API 상태가 제한될 경우 성공 화면을 임의로 꾸미지 않는다. 현재 구현 코드와 검증된 응답 구조를 이용한 설명용 캡처임을 표시한다.

## 검증 기준

- PDF가 정확히 12페이지인지 확인한다.
- 서비스 URL과 PDF 링크, PWA·Android·iPhone·PC 안내가 유지되는지 확인한다.
- 표지의 대형 배경 `담다`가 제거됐는지 확인한다.
- 최신 지도, 순간 카드, 장소 대표이미지, 지도 마커, 최신 하루방 화면이 포함됐는지 확인한다.
- 10페이지의 설명이 현재 하루방 기능과 일치하는지 코드·화면을 함께 대조한다.
- 12페이지 전체를 렌더링해 이미지 선명도, 잘림, 겹침, 캡션과 본문 가독성을 확인한다.
- 공모전 자료 폴더에는 검증된 최신 PDF만 제출용 파일명으로 저장한다.
