# 하루방 답변 가독성 디자인

## 목표

하루방의 긴 웹 조사 답변을 작은 채팅 패널에서도 빠르게 훑고 이해할 수 있게 만든다.

## 결정

- 답변은 Markdown으로 작성한다.
- 순서는 `한눈에 보기 → 장소별 추천 → 방문 팁`으로 제한한다.
- 장황한 조사 선언, 반복 경고, A/B/C/D 선택지는 제거한다.
- 추천은 검증된 고유 장소 5~6곳 이내로 제한하고, 불명확한 카테고리를 장소처럼 나열하지 않는다.
- 출처는 각 주장 가까이에 Markdown 링크로 표시한다.
- 프론트는 `react-markdown`과 `remark-gfm`으로 제목, 목록, 강조, 링크를 안전하게 렌더링한다.
- 사용자 메시지는 기존의 단순 텍스트 렌더링을 유지한다.

## 시각 구조

- 답변 말풍선은 패널 너비를 충분히 사용한다.
- 제목, 문단, 목록 사이에 명확한 수직 간격을 둔다.
- 장소 제목은 굵게, 세부 정보는 짧은 목록으로 표현한다.
- 링크는 밑줄과 명확한 포커스 상태를 제공하고 새 탭에서 연다.
- 출처·신뢰도 배지는 본문과 분리된 하단 메타 영역에 유지한다.

## 검증

- 백엔드 프롬프트 계약 테스트
- 프론트 타입 검사와 프로덕션 빌드
- 데스크톱과 모바일 채팅 패널에서 실제 긴 답변 시각 검증
