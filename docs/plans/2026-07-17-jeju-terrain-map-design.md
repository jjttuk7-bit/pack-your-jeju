# 제주 지역 선택 지도 입체 지형형 시각 고도화 설계

## 목표

지역 선택과 공공데이터 상태를 읽는 현재 기능은 유지하면서, 제주 지도가 첫 화면의 시각적 중심으로 느껴지도록 지형감과 선택 피드백을 강화한다. 외부 지도 이미지나 신규 SDK를 추가하지 않고 SVG와 CSS만 사용해 초기 로딩, PWA, 접근성 계약을 보존한다.

## 디자인 방향

선택한 방향은 **입체 지형형**이다. 실제 고도 데이터를 정밀하게 재현하는 지도가 아니라, 제주를 한눈에 인식하게 하는 절제된 지형 표현을 목표로 한다.

- 바다는 아이보리 단색 대신 청록·하늘색의 낮은 채도 수심 그라데이션으로 표현한다.
- 본섬 외곽에는 넓고 부드러운 그림자, 얇은 해안 하이라이트, 가까운 수면의 밝은 띠를 겹친다.
- 섬 중앙에는 한라산을 암시하는 방사형 음영과 3단 등고선을 배치한다.
- 행정구역 데이터 상태 색은 기존 의미를 유지하되 지형 레이어가 색을 가리지 않게 낮은 투명도로 사용한다.
- 선택된 지역은 주황색 외곽광, 미세한 상승 효과, 체크 마커로 다른 상태와 분명히 구분한다.

## 화면 구성

### 바다 레이어

지도 카드 배경에 세 단계의 시각 레이어를 둔다.

1. 위쪽은 옅은 안개빛, 아래쪽은 조금 깊은 청록빛인 수직 그라데이션
2. 섬을 따라 흐르는 두 개의 낮은 대비 파도·수심 등고선
3. 지도 가장자리의 부드러운 비네트와 빛 번짐

장식 레이어에는 `pointer-events: none`을 적용해 지역 터치 영역을 침범하지 않는다.

### 제주 본섬과 한라산

기존 `REGION_SHAPES` 경로를 합친 제주 전체 실루엣을 별도 장식 경로로 추가한다. 같은 실루엣을 여러 번 사용해 바다 그림자, 해안광, 외곽선을 만든다.

한라산은 본섬 중앙을 기준으로 한 방사형 그라데이션과 타원형 등고선으로 표현한다. 행정구역 경계를 가리는 사실적인 산 그림은 사용하지 않는다. 중앙부가 아주 약하게 높아 보이는 수준으로 제한한다.

### 지역 상태

- 기본: 현재 데이터 상태 색과 얇은 흰색 경계
- 호버·포커스: 명도 상승, 외곽선 강화, 지역명 배경 캡슐 표시
- 활성 지역: 청록색 안쪽 테두리와 명확한 라벨 강조
- 플랜 선택 지역: 주황색 외곽광과 체크 표시, 약한 상승 그림자
- 데이터 로딩: 회색 단색 대신 느린 광택 이동으로 조회 중임을 표현하되 `prefers-reduced-motion`에서는 정지

지역명 아래에는 공간이 허용되는 지역만 `후보 N` 또는 `확인 필요`처럼 짧은 상태를 표시한다. 지도에서 상세 수치를 과도하게 보여주지 않고 우측 패널의 설명을 보조하는 수준으로 제한한다.

## 상호작용

- 한 번 클릭 또는 Enter/Space: 현재 지역의 근거 패널 열기
- 우측 패널 버튼: 플랜 후보 선택·해제
- 기존 더블클릭 선택은 하위 호환으로 유지하되 주 안내 문구에서는 강조하지 않는다.
- 선택 시 지역 면 자체를 좌표 이동하지 않는다. SVG 필터와 시각적 그림자만 바꿔 인접 지역 경계가 벌어져 보이는 문제를 방지한다.
- 작은 화면에서는 상태 보조 라벨을 줄이고 기존 모바일 안내문을 지도 아래에 유지한다.

## 접근성

- 각 지역 경로의 `role="button"`, `tabIndex`, `aria-pressed`, 지역별 `aria-label`을 유지한다.
- 색상만으로 데이터 상태를 구분하지 않도록 범례 문구, 체크 표시, 보조 상태 텍스트를 함께 사용한다.
- 포커스 상태는 호버보다 강한 4px 외곽선으로 표시한다.
- 장식용 SVG 그룹은 `aria-hidden="true"` 또는 읽기 순서에서 제외한다.
- 애니메이션은 `prefers-reduced-motion`에서 비활성화한다.

## 구현 구조

`TrustMapDashboard.tsx` 안의 데이터·선택 로직은 변경하지 않고 지도 표현만 작은 컴포넌트와 헬퍼로 분리한다.

- `JejuSilhouetteMap`: 지도 카드와 상호작용 조립
- `JejuMapDefs`: 그라데이션·필터·클립 경로
- `JejuSeaLayer`: 수심, 파도, 외곽 비네트
- `JejuTerrainLayer`: 전체 실루엣, 해안광, 한라산 등고선
- `RegionStatusLabel`: 후보 수·확인 필요 보조 라벨

외부 이미지, 캔버스, WebGL, 실제 고도 데이터는 도입하지 않는다.

## 성능

- 현재 지연 로딩된 `TrustMapDashboard` 청크 안에서만 SVG 정의가 로드된다.
- 복잡한 blur 필터는 본섬 전체 그림자와 선택 지역에만 제한한다.
- 애니메이션은 `transform`, `opacity`, 필터 강도만 사용한다.
- 프로덕션 빌드에서 초기 진입 chunk 500KB 이하 계약을 유지한다.

## 테스트와 완료 기준

- 지도 장식 레이어가 지역 버튼보다 앞에서 이벤트를 가로채지 않는다.
- 모든 12개 지역이 키보드로 포커스되고 Enter/Space로 근거 패널을 열 수 있다.
- 선택 지역에 체크·선택광이 표시되고 `aria-pressed`가 유지된다.
- 데이터 상태 범례와 지역 보조 상태가 기존 API 결과만 사용한다.
- 모바일 안내문은 지도 아래, 데스크톱 안내문은 기존 위치를 유지한다.
- `prefers-reduced-motion` 대응 클래스가 존재한다.
- 전체 프론트 테스트, Node 회귀, TypeScript 검사, PWA 빌드가 통과한다.

## 비목표

- 실제 지형 고도나 도로를 정확히 재현하는 지도
- 지도의 확대·축소·드래그
- 외부 지도 SDK 또는 신규 이미지 에셋 추가
- 선택·공공데이터 API 계약 변경
- 우측 지역 설명 패널의 정보 구조 개편
