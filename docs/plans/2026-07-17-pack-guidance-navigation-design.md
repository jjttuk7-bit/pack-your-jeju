# 여행팩 진행 안내 내비게이션 설계

> 2026-07-18 변경: 아래의 데이터 기반 단계 판정은
> `2026-07-18-pack-journey-sequential-guidance-design.md`로 대체됐다.
> 화면 배치·접근성·기존 섹션 이동 원칙은 계속 유효하다.

## 목표

현재 팩 결과 화면의 풍부한 정보와 카드 구조를 유지하면서, 처음 방문한 사용자도 `후보 확인 → 플랜에 담기 → 일정 정리 → 다운로드·공유` 순서를 바로 이해하고 이동할 수 있게 한다.

## 핵심 원칙

- 기존 섹션의 순서, 기능, 데이터 흐름을 바꾸지 않는다.
- 콘텐츠를 잠그거나 강제로 한 단계씩 진행시키지 않는다.
- 사용자가 현재 어디까지 완료했으며 다음에 무엇을 하면 되는지만 알려준다.
- 하루방 패널과 충돌하지 않도록 우측 플로팅 UI를 추가하지 않는다.

## 상단 진행 내비게이션

팩 결과 화면 상단에 가로형 4단계 안내판을 배치한다.

1. 후보 고르기
2. 플랜에 담기
3. 일정 정리
4. 저장·공유

각 단계는 `완료`, `현재`, `다음` 상태를 가진다.

- 완료: 민트색 체크 표시와 완료 요약
- 현재: 감귤색 강조와 현재 해야 할 일
- 다음: 중립색으로 표시하되 클릭 가능

단계를 누르면 기존 화면의 해당 섹션으로 부드럽게 이동한다.

## 현재 행동 카드

진행판 아래에는 한 줄짜리 현재 행동 안내를 표시한다.

예시:

- `마음에 드는 장소를 플랜에 담아보세요.`
- `3곳을 담았어요. 이제 Day별 일정을 정리해보세요.`
- `일정이 준비됐어요. PDF로 저장하거나 함께 갈 사람에게 공유해보세요.`

안내 카드에는 현재 단계와 연결된 버튼 하나만 강조한다.

## 단계 판정

단계는 후보 수나 미리 채워진 Day 값으로 자동 완료하지 않는다. 사용자가 각 안내를 보고 후보 영역으로 이동하고, 플랜에 장소를 담고, 일정 값을 직접 편집한 행동을 순서대로 확인해 다음 단계로 전환한다. 상세 상태 전환은 후속 설계를 따른다.

## 반응형 동작

- 현재의 좁은 플랜 사이드바에서는 데스크톱과 모바일 모두 4단계를 2×2로 표시해 단계명이 잘리지 않게 한다.
- 향후 안내가 전체 폭 영역으로 이동할 때만 4단계를 한 줄로 확장한다.
- 진행판은 페이지 상단 콘텐츠에 포함하며 화면을 계속 가리는 고정 헤더로 만들지 않는다.

## 접근성

- 단계는 실제 버튼 또는 링크로 제공한다.
- 현재 단계는 `aria-current="step"`으로 표시한다.
- 색상뿐 아니라 체크 아이콘과 텍스트로 상태를 구분한다.
- 부드러운 스크롤을 사용하되 모션 감소 설정에서는 즉시 이동한다.
- 이동 후 대상 섹션 제목에 포커스를 전달해 키보드와 스크린리더 사용자가 위치를 알 수 있게 한다.

## 테스트

- 플랜 장소 수와 Day 정보에 따른 현재 단계 계산
- 단계 버튼의 `aria-current`와 완료 상태
- 각 단계가 기존 섹션 ID로 이동하는 계약
- 기존 팩 담기, 일정 편집, PDF, 공유 기능의 회귀 방지
- 1440px 및 390px에서 수평 넘침과 하루방 패널 충돌 확인

## 제외 범위

- 기존 섹션의 접기/펼치기
- 자동 일정 생성 방식 변경
- PDF 및 공유 기능 변경
- 첫 방문 모달 투어
- 새로운 서버 API 또는 저장 상태
