# Haruban General Question Quality Design

## Goal

하루방이 모든 질문을 장소 추천 검색으로 보내지 않고, 제주 일반 지식·여행 계획 질문에는 자연스럽게 답하도록 만든다.

## Problem

현재 하루방은 장소 추천, 웹검색, 일반 여행 상담이 같은 입구를 공유한다. 그래서 "제주공항은 어느 지역에 있어?" 같은 질문도 장소 상세 또는 웹검색 흐름으로 새기 쉽다. 웹검색이 실패하거나 API 키가 없으면 사용자는 단순 질문에도 "공공데이터 기준으로 확인되지 않는다"는 답을 받는다.

## Design

하루방 앞단에 결정적 일반 질문 경로를 둔다.

1. 특정 장소 추천·목록·운영시간·주소·교통·수정요청·리뷰 검증은 기존 도구 경로를 유지한다.
2. "제주공항은 어느 지역", "제주는 처음", "권역 추천", "공항 근처 동선"처럼 공통적인 제주 여행 상담은 검색 전 템플릿 답변으로 처리한다.
3. 템플릿은 새 장소명, 운영시간, 수치성 사실을 만들지 않는다. 단, 안정적인 지역 설명과 서비스 사용 안내는 제공한다.
4. 답변 마지막에는 사용자가 지역·순간을 선택하면 공공데이터 후보로 좁힐 수 있음을 자연스럽게 연결한다.

## Initial Coverage

- 제주공항 위치/권역 질문
- 제주 첫 여행 권역 선택 질문
- 제주시를 처음 볼 때의 큰 방향 질문
- 공항 도착·출발일 동선 질문

## Non-goals

- 웹검색 대체 엔진을 새로 만들지 않는다.
- 장소 추천 품질 랭킹을 이번 변경에서 바꾸지 않는다.
- 운영시간·요금·혼잡도 같은 최신 정보는 템플릿에서 말하지 않는다.
