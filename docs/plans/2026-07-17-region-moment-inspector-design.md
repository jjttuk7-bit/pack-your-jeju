# 지역별 순간 조합 탐색 패널 설계

## 배경

현재 `TrustMapDashboard`의 오른쪽 패널은 활성 지역 하나에 대해 선택한 순간을 최대 4개만 보여준다. 구현에도 `momentStories.slice(0, 4)` 제한이 있어 다섯 번째 이후 순간은 화면에 나타나지 않는다. 여러 지역을 선택하면 지역 이름은 합산 요약에 표시되지만, 사용자가 각 지역과 순간의 조합을 빠짐없이 확인할 수 있는 탐색 구조는 없다.

사용법 안내만 추가하면 숨겨진 조합의 존재와 현재 확인 위치를 계속 기억해야 한다. 따라서 긴 카드를 모두 나열하는 방식 대신, 선택 전체를 요약하고 지역과 순간을 차례로 고른 뒤 한 조합의 상세 근거를 확인하는 마스터-디테일 구조가 필요하다.

## 목표

- 선택한 순간이 4개를 넘어도 모든 순간에 접근할 수 있게 한다.
- 여러 지역과 여러 순간의 조합 수와 확인 진행률을 한눈에 보여준다.
- `플랜 후보로 선택한 지역`과 `현재 근거를 확인하는 지역`을 구분한다.
- 한 번에 상세 카드 하나만 보여 정보 밀도를 관리한다.
- 사용자가 지도, 지역 탭, 순간 칩 중 어느 경로를 사용해도 같은 활성 조합으로 연결한다.
- 데스크톱과 모바일에서 동일한 정보 구조를 유지한다.

## 비목표

- 지역과 순간의 모든 조합을 서버에 저장하지 않는다.
- 사용자가 조합을 확인했다는 상태를 계정이나 플랜에 영구 저장하지 않는다.
- 후보 장소 조회, 제주팩 생성 API, 공공데이터 점수 산정 방식을 변경하지 않는다.
- 자동으로 지역이나 순간 선택을 해제하거나 플랜을 수정하지 않는다.

## 핵심 정보 구조

오른쪽 패널을 다음 세 구역으로 나눈다.

1. **고정 요약**
   - `3개 지역 · 8개 순간 · 총 24개 조합`
   - `확인 5 / 24`
   - 짧은 안내: `지역과 순간을 눌러 조합별 근거를 확인하세요.`
2. **조합 탐색**
   - 선택 지역을 탭 또는 칩으로 표시한다.
   - 활성 지역 아래에 선택한 모든 순간을 칩으로 표시한다.
   - 순간 칩은 `확인함`, `현재 확인`, `미확인` 상태를 구분한다.
   - `미확인만 보기`를 켜면 아직 열지 않은 순간만 남긴다.
3. **상세와 행동**
   - 활성 지역과 활성 순간의 상세 근거 카드 하나를 표시한다.
   - `이전`, `다음`, `현재 순번 / 전체 조합 수`로 순차 탐색한다.
   - 지역 선택/해제와 `제주팩 만들기` 버튼은 패널 하단에 유지한다.

## 지역별 묶음

여러 지역이 선택되면 지역을 1차 탐색 단위로 사용한다.

```text
선택 현황
3개 지역 · 8개 순간 · 총 24개 조합
확인 5 / 24

[한림 8] [구좌 8] [서귀포 8]

한림
[오름 ✓] [바다 ✓] [노을 ●] [카페 ○] [맛집 ○] ...

한림 × 노을 명소에서 멍 때리기
[이전] 3 / 24 [다음]
```

- 지역 탭의 숫자는 그 지역에서 확인할 수 있는 선택 순간 수다.
- 지도에서 지역 면을 한 번 누르면 해당 지역을 활성화하지만 선택 여부는 바꾸지 않는다.
- 지도에서 기존 방식대로 선택 동작을 수행하면 플랜 후보 포함 여부만 바뀌고, 해당 지역은 활성 지역이 된다.
- 선택하지 않은 지역을 단순 확인 중인 경우에는 해당 지역 하나의 순간 상세를 보여주되 전체 조합 진행률에는 포함하지 않는다.

## 순간 상태

- **현재 확인 `●`**: 활성 지역과 활성 순간의 조합
- **확인함 `✓`**: 이번 화면에서 사용자가 한 번 이상 상세를 연 조합
- **미확인 `○`**: 아직 상세를 열지 않은 조합

확인 상태는 브라우저 메모리의 UI 상태로만 유지한다. 지역 또는 순간을 선택 해제하면 해당 조합은 진행률 계산에서 빠진다. 다시 선택했을 때 같은 화면 세션 안에서는 기존 확인 상태를 재사용할 수 있다.

## 선택 변화 규칙

- 순간을 새로 선택하면 그 순간을 활성 순간으로 삼고 현재 활성 지역의 상세를 연다.
- 활성 순간을 선택 해제하면 남은 첫 번째 선택 순간을 활성화한다.
- 선택 순간이 없으면 상세 대신 빈 상태와 선택 안내를 보여준다.
- 지역을 새로 선택하면 해당 지역을 활성화하고 현재 활성 순간을 유지한다.
- 활성 지역을 선택 해제하면 남은 선택 지역의 첫 번째 지역으로 이동한다.
- 선택 지역이 없으면 현재 지도에서 확인 중인 지역을 유지하되 제주팩 생성 버튼은 비활성화한다.

## 이전·다음 탐색

전체 조합 순서는 `선택 지역 순서 × 선택 순간 순서`로 결정한다. 예를 들어 지역이 `한림, 구좌`, 순간이 `오름, 바다, 카페`라면 다음 순서로 이동한다.

```text
한림×오름 → 한림×바다 → 한림×카페
→ 구좌×오름 → 구좌×바다 → 구좌×카페
```

이전·다음으로 연 조합도 `확인함`으로 기록한다. 마지막 조합에서 다음을 누르면 첫 조합으로 순환하지 않고 버튼을 비활성화한다.

## 레이아웃과 반응형

- 데스크톱에서는 오른쪽 패널 요약과 하단 행동을 고정하고 중간 탐색 영역만 내부 스크롤한다.
- 순간 칩은 줄바꿈하며 12개 모두 렌더링한다. 칩 영역이 길어지면 접기·펼치기를 제공하되 숨겨진 개수를 명시한다.
- 모바일에서는 오른쪽 패널을 본문 아래의 전체 폭 영역으로 유지하고, 향후 지도 바텀시트 전환이 가능하도록 탐색 컴포넌트를 독립시킨다.
- 가로 스크롤만으로 숨겨진 항목을 찾아야 하는 구조는 사용하지 않는다.

## 접근성

- 지역 목록은 `aria-label="선택 지역 조합"`을 가진 내비게이션으로 제공한다.
- 지역 칩은 `aria-current`로 활성 지역을 표시한다.
- 순간 칩은 `aria-pressed`와 상태 텍스트를 함께 제공한다.
- 진행률은 텍스트와 `<progress>`를 함께 사용한다.
- 이전·다음 버튼의 비활성 상태를 실제 `disabled` 속성으로 전달한다.
- 키보드 포커스 순서는 요약 → 지역 → 순간 → 상세 → 이전/다음 → 주요 행동 순서를 따른다.

## 오류와 빈 상태

- 지역 근거 API가 실패해도 선택 구조는 유지하고 해당 조합 상세에 `근거를 불러오지 못했습니다`를 표시한다.
- 공공데이터가 부족한 조합은 숨기지 않고 기존 `데이터 부족` 상태와 확인 필요 문구를 유지한다.
- 선택 지역 또는 순간이 없으면 총 조합 수를 0으로 표시하고 다음 행동을 안내한다.

## 검증 기준

- 1개 지역과 10개 순간을 선택했을 때 10개 순간 칩을 모두 키보드와 포인터로 열 수 있다.
- 3개 지역과 10개 순간을 선택했을 때 총 30개 조합과 확인 진행률이 정확하다.
- 지도에서 다른 지역을 확인해도 기존 플랜 후보 선택이 사라지지 않는다.
- 다섯 번째 이후 순간에도 기존과 동일한 근거 문구가 표시된다.
- 데스크톱 패널에서 상세 카드가 하나만 렌더링되고 주요 행동 버튼이 접근 가능하다.

## 결정

여러 지역과 여러 순간은 **지역별로 묶은 마스터-디테일 패널**로 탐색한다. 사용법 팝업이나 4개 제한을 유지하지 않으며, 모든 선택 조합에 접근 가능한 명시적 탐색 경로를 제공한다.
