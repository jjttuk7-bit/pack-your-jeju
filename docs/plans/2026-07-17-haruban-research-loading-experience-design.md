# 하루방 리서치 로딩 경험 설계

## 목표

하루방의 웹 리서치가 20초 안팎 걸릴 때 단순 스피너 대신 조사 과정과 제주 여행 팁을 보여줘 체감 대기시간을 줄인다. 실제 서버 진행률을 아는 것처럼 과장하지 않으면서도, 사용자가 하루방이 근거를 찾고 비교하는 서비스라는 점을 자연스럽게 이해하게 한다.

## 선택된 방향

- 캐릭터성과 조사 과정을 결합한 균형형
- 요청 직후에는 작은 말풍선 카드
- 8초 이상 걸리면 채팅 폭을 활용하는 탐험 카드로 확장
- 조사 안내와 짧은 제주 여행 팁을 번갈아 표시
- 예상 남은 시간이나 가짜 진행률 대신 실제 경과 시간만 표시
- 취소 버튼은 제공하지 않음
- 채팅창을 닫아도 진행 중인 요청은 유지

## 사용자 경험

### 0~8초: 작은 조사 말풍선

사용자 질문 바로 아래에 하루방 캐릭터와 한 줄 안내를 표시한다.

예시:

> 질문에 담긴 여행 조건을 살펴보고 있어요

카드는 기존 답변 말풍선보다 조금 밝은 크림색 배경을 사용한다. 하루방 캐릭터는 가볍게 고개를 움직이고, 작은 점 세 개가 순서대로 나타난다. 짧게 끝나는 요청은 이 상태에서 바로 답변으로 교체된다.

### 8초 이후: 제주 탐험 카드

같은 카드가 새 요소를 드러내며 자연스럽게 확장된다. 별도의 모달이나 전체 화면을 띄우지 않는다.

구성:

1. 하루방 캐릭터와 현재 경과 시간
2. 조사 안내 문구
3. 제주 지형을 연상시키는 짧은 이동 경로 애니메이션
4. `기다리는 동안 제주 한 조각` 팁 영역

경과 시간 예시:

> 12초째 제주 곳곳의 근거를 살펴보고 있어요

경과 시간은 요청을 보낸 시점부터 실제로 계산한다. 완료율이나 남은 시간은 표시하지 않는다.

## 조사 안내 문구

안내 문구는 실제 서버의 세부 이벤트가 아니라 하루방이 답변을 만들 때 따르는 원칙을 설명하는 표현이다. 따라서 `완료`, `80%`처럼 특정 단계가 끝났다고 단정하지 않는다.

4초 간격으로 다음 문구를 순환한다.

1. 질문에 담긴 지역과 여행 취향을 읽고 있어요
2. 공식·플랫폼·경험 출처를 폭넓게 살펴보고 있어요
3. 서로 다른 정보를 나란히 비교하고 있어요
4. 확인한 근거를 여행자가 읽기 쉽게 정리하고 있어요

표시 영역에 `조사 중 안내`라는 작은 라벨을 붙여 실제 진행률 표시와 구분한다.

## 제주 여행 팁

확장 카드에서 조사 안내와 함께 5초 간격으로 짧은 팁을 순환한다. 팁은 특정 장소의 최신 운영 사실을 단정하지 않고 오래 유지되는 일반 여행 지식으로 제한한다.

초기 팁:

- 제주에서는 같은 지역도 해안과 중산간의 날씨가 다를 수 있어요.
- 바람이 강한 날에는 우산보다 가벼운 우비가 편할 때가 많아요.
- 인기 장소는 운영시간뿐 아니라 마지막 입장 시간도 함께 확인해 보세요.
- 이동 시간을 잡을 때는 지도 거리보다 제주 도로의 실제 흐름을 여유 있게 보세요.
- 여행 중 발견한 변경 정보는 다음 여행자의 더 나은 근거가 될 수 있어요.

팁은 하루방의 최종 추천이나 검색 결과와 혼동되지 않도록 `기다리는 동안 제주 한 조각` 제목 아래 별도 색상으로 표시한다.

## 화면과 모션

### 시각 방향

- 기존 크림·감귤·민트 색상과 현무암 톤을 유지한다.
- 기존 `HarubangMark`를 재사용한다.
- 제주 섬을 직접 정교하게 그리기보다 점과 곡선으로 된 작은 탐험 경로를 사용한다.
- 카드 확장은 높이가 갑자기 튀지 않도록 `motion`의 레이아웃 전환을 사용한다.

### 모션

- 하루방: 2~3px 범위의 느린 상하 움직임
- 탐험 점: 곡선 경로를 따라 이동
- 조사 문구와 팁: 짧은 페이드 전환
- 점 세 개: 순차적인 투명도 변화

`prefers-reduced-motion` 사용자는 캐릭터와 경로 이동을 멈추고 문구만 즉시 교체한다.

## 컴포넌트와 상태

새 컴포넌트 `HarubanResearchLoading`을 `HarubanChat.tsx`와 같은 컴포넌트 영역에 둔다.

입력값:

- `startedAt`: 요청 시작 시각
- `active`: 요청 진행 여부

내부 상태:

- `elapsedSeconds`: 1초마다 갱신
- `guideIndex`: 4초마다 순환
- `tipIndex`: 5초마다 순환
- `expanded`: `elapsedSeconds >= 8`

`HarubanChat`의 기존 `loading`이 `true`가 되는 순간 요청 시작 시각을 저장한다. 응답 또는 오류가 도착하면 로딩 컴포넌트를 제거하고 타이머를 정리한다.

채팅창의 `open` 상태와 요청 상태는 분리한다. 사용자가 패널을 닫아도 `requestHarubanChat` 호출과 타이머는 유지되며, 다시 열면 현재 경과 시간에 맞는 카드가 보인다.

## 오류와 느린 요청 처리

- 응답 실패 시 로딩 카드는 제거하고 기존 오류 안내를 표시한다.
- 30초 이상이어도 오류로 단정하지 않고 경과 시간과 조사 안내를 계속 표시한다.
- `조사 그만하기` 버튼은 제공하지 않는다.
- 요청이 완료되면 카드가 사라지고 기존 답변 말풍선과 장소 선택 카드가 그대로 나타난다.
- 여러 요청을 동시에 보내는 기존 제한을 유지한다.

## 접근성

- 로딩 영역은 `role="status"`와 `aria-live="polite"`를 사용한다.
- 매초 경과 시간 갱신을 스크린리더가 모두 읽지 않도록, 접근성용 문구는 주요 전환 시점에만 바꾼다.
- 장식용 경로와 점은 `aria-hidden="true"`로 처리한다.
- 텍스트 대비는 기존 서비스 색상 대비를 유지한다.
- 모션 감소 환경에서는 반복 이동 애니메이션을 제거한다.

## 테스트

1. 소스 계약 테스트
   - 작은 카드 문구와 8초 확장 기준이 존재한다.
   - 조사 안내 4종과 제주 팁 목록이 존재한다.
   - 경과 시간 표시가 있고 예상 남은 시간·진행률·취소 버튼은 없다.
   - `role="status"`와 모션 감소 처리가 존재한다.

2. TypeScript 검사
   - `npm run lint`

3. 프로덕션 빌드
   - `npm run build`

4. 수동 검증
   - 8초 이전에는 작은 카드만 보인다.
   - 8초 이후 카드가 확장되고 경과 시간·조사 안내·팁이 보인다.
   - 패널을 닫았다 다시 열어도 요청이 중단되지 않는다.
   - 답변 도착 시 로딩 카드가 사라지고 기존 답변과 장소 체크 UI가 나타난다.
   - 모바일 폭에서 입력창이나 메시지 영역을 가리지 않는다.
   - 모션 감소 설정에서 반복 애니메이션이 멈춘다.

## 비범위

- 서버의 실시간 검색 단계 스트리밍
- 예상 남은 시간 계산
- 퍼센트 진행률
- 요청 취소 API
- 사용자별 팁 설정
- 검색 결과가 오기 전 임시 장소 추천
