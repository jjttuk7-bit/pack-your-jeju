# Haruban Web-First Routing Design

## Problem

`제주시에서 제일 첫번째로 방문하면 좋은 곳` 같은 추천 질문이 특정 장소 상세질문으로 잘못 분류되어 OpenAI 웹검색 전에 내부 공공데이터 조회로 빠졌다. 그 결과 장소명이 아닌 문장 전체를 DB에서 찾고 `확인되지 않았습니다`라는 낮은 가치의 답변을 반환했다.

## Decision

- 첫 방문지, 좋은 곳, 방문 추천 표현은 지역 추천 질문으로 분류한다.
- 추천·맛집·카페·명소 질문은 Responses API의 `web_search`를 우선 실행한다.
- 첫 검색에서 인용 가능한 출처를 얻지 못하면 선택 지역의 행정구역명과 공식·지도·최근 후기 관점을 포함해 한 번 재검색한다.
- 두 검색이 모두 실패한 경우에만 웹검색 실패 범위를 투명하게 알린다.
- 내부 공공데이터는 웹 답변 뒤 교차확인 자료로만 유지한다.

## Verification

- 첨부 화면의 질문이 `web_search_jeju`로 라우팅되는 회귀 테스트를 추가한다.
- 첫 검색 실패 뒤 두 번째 검색이 실행되는 테스트를 추가한다.
- 하루방 전체 테스트로 장소 상세, 교통, 날씨, 명시적 공공데이터 질문의 기존 동작을 확인한다.
