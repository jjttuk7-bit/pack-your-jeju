# 하루방 플랜 조합 설계

## 목적

여행 장소를 담아두었지만 Day·시간대·방문 순서를 직접 구성하기 어려운 사용자를 위해, 하루방이 현재 날씨와 동선 정보를 반영한 **편집 가능한 여행 초안**을 만든다.

이 기능은 선택 사항이다. 사용자가 직접 구성한 `내 여행플랜`을 덮어쓰거나 자동 확정하지 않는다.

## 제품 원칙

1. 사용자가 담은 장소만 사용한다. 하루방이 장소를 임의로 추가하거나 삭제하지 않는다.
2. 공공데이터, 하루방 웹검색, 직접 추가의 출처 구분을 유지한다.
3. 사용자가 고정한 시간과 예약 일정은 이동하지 않는다.
4. 날씨와 이동시간은 기존 확인 기능의 실제 결과만 사용한다.
5. 확인하지 못한 좌표·날씨·영업 정보는 추측하지 않고 `확인 필요`로 남긴다.
6. 결과는 `최적 플랜`이 아니라 `하루방 추천 초안`으로 표현한다.
7. 최종 Day·순서·메모와 PDF 다운로드 여부는 사용자가 결정한다.

## 사용자 흐름

```text
내 여행플랜에 장소 담기
  -> 하루방 플랜 조합 선택
  -> 날씨 새로 확인
  -> 고정·예약 일정 보호
  -> 날씨에 맞는 Day·시간대 초안 구성
  -> 좌표가 확인된 장소의 동선 계산
  -> Day 안에서 방문 순서 조정
  -> 하루방 추천 초안 완성 알림
  -> PDF 편집기에서 비교·수정
  -> PDF 다운로드 또는 플랜 공유
```

## 화면 설계

### 조합 버튼

`내 여행플랜에 담은 것` 카드의 제목과 설명 아래에 전체 너비의 보조 버튼을 둔다.

- 제목: `하루방 플랜 조합`
- 설명: `담아둔 장소를 날씨와 동선에 맞춰 일정 초안으로 만들어요.`
- 장소가 없으면 비활성화한다.
- 실행 중에는 중복 클릭을 막는다.

### 진행 상태

단순 스피너 대신 현재 작업을 알려준다.

1. `여행 날짜와 장소를 정리하고 있어요`
2. `날씨를 확인하고 있어요`
3. `가까운 장소끼리 동선을 살펴보고 있어요`
4. `Day별 추천 초안을 마무리하고 있어요`

### 완료 상태

조합이 끝나면 원본 목록 위에 결과 카드를 표시한다.

- `하루방이 4개 장소로 2일 여행 초안을 만들었어요.`
- `날씨 반영`, `동선 반영`, `부분 확인` 상태 배지
- `초안 확인하기`
- `다시 조합`
- 원본 플랜이 바뀌면 기존 초안은 폐기하고 다시 조합하도록 알린다.

## 데이터 모델

`selectedPlanItems`는 원본 플랜으로 유지한다.

별도 `HarubanPlanDraft`를 둔다.

```ts
interface HarubanPlanDraft {
  title: string;
  items: PlanPdfDraftItem[];
  sourcePlanFingerprint: string;
  createdAt: string;
  weather: {
    status: 'applied' | 'unchanged' | 'partial' | 'unavailable';
    headline: string;
    checkedAt?: string | null;
  };
  route: {
    status: 'applied' | 'unchanged' | 'partial' | 'unavailable';
    headline: string;
    checkedAt?: string | null;
  };
  reasonsByItemId: Record<string, string[]>;
  warnings: string[];
}
```

초안은 우선 `PackingDashboard` 세션 상태로 관리한다. 원본 플랜 fingerprint가 달라지면 오래된 초안으로 표시하지 않고 폐기한다.

## 조합 방식

V1은 검증 가능한 기존 함수를 오케스트레이션한다.

1. `schedulePlanItemsForWeather`로 원본을 복사해 기본 Day·시간대를 채운다.
2. `/weather/report`를 새로 호출한다.
3. 적용 가능한 날씨 제안을 **복사본에만** 반영한다.
4. 좌표가 있는 장소만 `/route/plan`으로 계산한다.
5. 적용 가능한 동선 제안을 복사본에만 반영한다.
6. 좌표가 없거나 지원하지 않는 지역의 장소는 기존 배치를 유지하고 경고를 붙인다.
7. 결과를 `PlanPdfDraft`로 변환해 PDF 편집기로 전달한다.

LLM이 날씨, 거리, 운영시간을 계산하지 않는다. 하루방이라는 이름은 검증된 도구들을 순서대로 사용해 초안을 조합하는 제품 역할을 뜻한다. 외부 호출이 실패해도 기본 Day 초안과 구체적인 제한 상태를 반환한다.

## PDF 편집기와 공유

하루방 초안에서 PDF 편집기를 열면:

- 상단에 `하루방 추천 초안` 표시
- 날씨·동선 반영 상태와 확인하지 못한 항목 표시
- 장소별 배치 이유 표시
- 기존 Day 변경, 순서 변경, 메모 편집 기능 유지
- 편집 내용은 현재 초안 상태에 반영

하루방 초안이 활성화된 동안 `플랜 공유`는 원본 대신 현재 초안의 Day·순서를 사용한다. 사용자가 `내 플랜으로 돌아가기`를 선택하면 다시 원본을 사용한다.

## 실패 처리

- 장소 0개: 버튼 비활성화
- 장소 1개: 날씨만 확인하고 동선은 `비교할 장소 없음`
- 날씨 실패: 기존 배치를 유지하고 `날씨 확인 실패`
- 좌표 부족: 확인 가능한 장소만 동선 계산, 나머지는 `동선 미확인`
- 동선 API 실패: 현재 순서를 유지하고 실패 이유 표시
- 조합 중 원본 변경: 진행 결과를 적용하지 않고 다시 조합하도록 안내

## 검증

- 원본 `selectedPlanItems`가 변경되지 않는지 확인
- 고정·예약 일정이 유지되는지 확인
- 날씨 제안과 동선 제안이 복사본에만 적용되는지 확인
- 부분 실패에서도 편집 가능한 초안이 생성되는지 확인
- 새 초안이 PDF 편집기 초기값과 공유 텍스트에 사용되는지 확인
- 원본 변경 시 오래된 초안이 폐기되는지 확인
