# 하루방 범용 라우팅과 세션 문맥 설계

## 목표

예측 가능한 질문 문장을 계속 추가하는 방식에서 벗어나, 처음 보는 표현의 제주 질문도 근거 필요 여부에 따라 안전하게 조사하고 현재 채팅 세션의 핵심 문맥을 이어받는다.

## 설계 원칙

- 검색하지 않아도 안전한 질문만 명시적으로 분리한다.
- 분류하기 어려운 제주 관련 질문은 웹 조사를 기본값으로 둔다.
- 짧은 후속 질문은 직전 지역·질문·후보를 복원한 뒤 조사한다.
- 가리키는 대상이 없으면 추측하지 않고 짧게 확인한다.
- 제주 여행 범위 밖의 질문은 제주 전용 에이전트의 범위를 안내한다.
- 문맥은 현재 열린 채팅에서만 유지하며 브라우저 새로고침 시 초기화한다.

## 검토한 접근

### 키워드 계속 추가

구현은 빠르지만 새로운 표현이 계속 누락된다. 기존의 정확한 표현 회귀 테스트는 유지하되 기본 전략으로 사용하지 않는다.

### 안전한 무검색 경로 + 제주 질문 웹 조사 기본값

선택한 방식이다. 인사, 에이전트 사용법, 일반적인 일정 구성법처럼 사실 검증이 필요하지 않은 질문만 검색 없이 처리한다. 그 외 제주 문맥의 질문은 표현을 예상하지 못했더라도 웹 조사로 보낸다.

### LLM 의도 분류

표현 대응력은 높지만 모델 장애 시 라우팅 자체가 중단되고 추가 비용과 지연이 발생한다. 핵심 라우팅에는 사용하지 않는다.

## 라우팅 구조

```text
사용자 질문
  -> 명확한 인사·에이전트 사용법인가? -> 결정론적 답변
  -> 명확한 일반 여행 상담인가? -> 일반 상담 답변
  -> 제주 외 질문이 명확한가? -> 제주 여행 범위 안내
  -> 짧은 후속 질문인가?
       -> 세션 대상 있음 -> 문맥을 붙여 웹 조사
       -> 세션 대상 없음 -> 한 문장 확인 질문
  -> 나머지 제주 문맥 질문 -> 웹 조사
```

제주 문맥은 현재 질문의 제주 지역·장소 표현뿐 아니라 선택된 폼 지역과 세션 문맥의 활성 지역·후보를 포함한다.

## 세션 문맥

프런트의 React 상태로 다음 구조만 유지한다.

- `last_user_question`: 마지막 사용자 질문
- `last_research_query`: 마지막 웹 조사 검색어
- `active_regions`: 현재 활성 지역
- `active_place_names`: 직전 답변의 활성 후보
- `shown_place_names`: 현재 세션에서 이미 제시한 후보
- `excluded_constraints`: 사용자가 제외를 요청한 원문 조건
- `web_research_active`: 후속 질문에서 웹 조사 의도를 이어갈지 여부

전체 화면 대화는 기존처럼 `entries`에 유지한다. API 요청에는 최근 24개 메시지와 구조화된 `conversation_state`를 함께 보낸다. 서버나 로컬 저장소에는 세션 문맥을 영속화하지 않는다.

## 데이터 흐름

1. 프런트가 사용자 질문과 현재 세션 문맥을 `/agent/chat`에 보낸다.
2. API는 메시지, 폼 상태, 세션 문맥을 함께 라우팅과 검색 문맥에 사용한다.
3. 응답의 검색 도구 추적과 구조화 장소 후보를 이용해 프런트가 세션 문맥을 갱신한다.
4. 다음 요청에서 갱신된 문맥을 다시 전송한다.
5. 새로고침 또는 컴포넌트 재생성 시 문맥은 초기화된다.

## 실패 처리

- 웹 검색 성공 후 최종 모델 실패: 기존 검색 결과 기반 폴백
- 웹 검색 실패: 확인하지 못한 범위를 투명하게 안내
- 모델 미가용 + 조사 근거 있음: 조사 근거 기반 폴백
- 모델 미가용 + 조사 근거 없음: 제한 상태 안내
- 모호한 후속 질문 + 활성 대상 없음: 대상 확인 질문
- 제주 외 질문: 외부 사실을 추측하지 않고 제주 여행 지원 범위 안내

## 테스트 전략

개별 문장뿐 아니라 표현군을 테이블 기반으로 검증한다.

- 지역 설명: `어떤 곳`, `뭐 하는 데`, `분위기`, `특징`
- 이유·유래: `왜 유명`, `이름 유래`, `알려진 이유`
- 후속 질문: `거기는?`, `왜?`, `그중 다른 곳`
- 제외 후 재추천: 후보명 제외, `그건 빼줘`, `다른 곳`
- 무검색: 인사, 에이전트 사용법, 일반 일정 구성
- 범위 밖: 명확한 비제주 지역 또는 비여행 최신 정보
- 모호성: 활성 대상 없는 짧은 지시어
- 장애: 검색 실패, 모델 실패, 검색 성공 후 모델 실패

프런트 테스트는 세션 문맥의 누적, 중복 제거, 상한, 새로고침 비영속성을 순수 함수 수준에서 검증한다.
