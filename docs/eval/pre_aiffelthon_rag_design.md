# 제주를 담다 RAG 설계 요약

생성일: 2026-07-08

## 한 줄 정의

제주를 담다는 GPT-5-mini가 장소를 생성하는 서비스가 아니라, 공공데이터와 사용자 신호를 검색·검증한 뒤 GPT-5-mini가 그 근거를 읽기 쉬운 언어로 조립하는 **근거 우선 RAG 서비스**다.

## 파이프라인

```text
사용자 입력
  지역 · 기간 · 동행자 · 목적 · 순간카드
        ↓
검색 필터 생성
  LLM 없이 구조화 필터로 변환
        ↓
근거 검색
  비짓제주 · 수정요청 · 교통/주차 · 기상청 · 방문 피드백
        ↓
신뢰도 계산
  공공데이터 30 + 사용자 조건 20 + 날씨 15 + 이동 10
  + 운영 정보 10 + 방문 신호 10 + 최신성 5
        ↓
GPT-5-mini 조립
  검색된 후보와 점수만 요약, 새 장소명/주소 생성 금지
        ↓
사용자 결과
  추천 카드 · 확인 필요 항목 · fallback_reason · 여행플랜 · 리뷰 검증
```

## GPT-5-mini 사용 위치

| 위치 | 역할 | 환각 방지 |
|---|---|---|
| 팩 소개 문구 | 조회된 후보와 부족한 조합을 자연어로 요약 | 목록에 없는 장소명·수치·시간 생성 금지 |
| 하루방 에이전트 | 현재 폼과 플랜 상태를 읽고 다음 행동을 안내 | 장소·주소는 DB 후보 안에서만 발화 |
| 리뷰 검증 | 사용자 리뷰 문장을 claim으로 분리하고 근거와 대조 | 검증 결과는 DB/수정요청/fallback_reason에 종속 |

## 성능 향상 시도

- 구조화 폼 입력으로 검색 범위를 먼저 좁혀 Self-Query 의존도를 줄임
- 공공데이터 후보 외 장소를 생성하지 않는 프롬프트 가드 적용
- fallback_reason을 4분기로 분리해 “정보 없음”을 뭉뚱그리지 않음
- 수정요청·방문 피드백·기상청 예보 신호를 trust_score에 반영
- 골든셋 게이트로 verified/fallback/badge 정확도를 반복 측정

## 발표용 핵심 문장

> GPT-5-mini는 답을 만드는 엔진이 아니라, 이미 검색·검증된 근거를 사용자 언어로 조립하는 마지막 단계입니다. 제주를 담다의 신뢰도는 모델 감이 아니라 원본 데이터와의 대조, fallback_reason, trust_score_breakdown으로 설명됩니다.
