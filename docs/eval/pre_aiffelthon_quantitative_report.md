# 제주를 담다 정량 평가 리포트

생성일: 2026-07-07

## 목적

프리아이펠 평가 항목 중 **신뢰성·설득력**을 증명하기 위해, 제주를 담다의 RAG/검증 파이프라인을 골든셋으로 점검했다. 이 평가는 모델이 그럴듯한 장소를 생성하는지가 아니라, 원본·출처 데이터와 대조해 **근거 있는 후보, 확인 필요 항목, 확인 불가 항목을 정확히 분리하는지**를 본다.

## 평가 방식

- 실행 명령: `python -m packages.eval.run --out docs/eval`
- 평가 케이스: 12개
- 대상 흐름: `/pack`, `/verify`
- 검증 기준:
  - Verified Precision: 근거 있는 후보가 기대 조건을 만족하는 비율
  - Fallback Accuracy: 확인 불가/검색 실패/반증 케이스의 reason 분류 정확도
  - Badge Accuracy: 주의/반증/수정요청 배지 판정 정확도

## 기본 GPT 대비 차이

| 비교 항목 | 기본 GPT 추천 | 제주를 담다 |
|---|---|---|
| 장소명 생성 | 모델이 그럴듯한 장소를 말할 수 있음 | DB 조회 후보만 사용 |
| 출처 제시 | 출처가 없거나 사후 확인 필요 | 비짓제주·수정요청·교통·기상청 신호를 구조화 |
| 정보 부족 대응 | 빈칸을 자연어로 채울 위험 | `coverage_gap`으로 분리하고 “공공데이터 기준 확인되지 않음”이라고 표시 |
| 반증 처리 | 폐업/이전 여부를 놓칠 수 있음 | 수정요청·tombstone·valid_until을 배지와 점수에 반영 |
| 날씨 반영 | 일반 조언 수준 | KMA API허브 예보 신호를 야외 장소 신뢰도에 반영 |
| 방문 후 개선 | 일회성 답변 | 방문 피드백을 신뢰도 변화와 수정요청 큐로 연결 |

## 발표에서 강조할 문장

> 제주를 담다는 “좋아 보이는 제주 여행지”를 만드는 서비스가 아니라, 공공데이터·기상청·수정요청·방문 피드백을 대조해 **믿을 수 있는 후보와 아직 확인이 필요한 후보를 분리하는 신뢰도 분석 서비스**입니다.
